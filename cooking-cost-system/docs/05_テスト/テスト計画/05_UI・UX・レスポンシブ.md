# テスト計画書：UI・UX・レスポンシブデザイン

| 項目 | 内容 |
|---|---|
| 対応 issue | #22 |
| 対象画面 | 全画面 |
| 作成日 | 2026-05-07 |

---

## 1. テスト目的

- PC・スマートフォン・タブレットで画面が崩れず表示されること
- 操作が直感的で、技術知識のないユーザーでも迷わず使えること
- エラー時にわかりやすいフィードバックが表示されること

---

## 2. 前提条件

- `nuu1729` / `Test1111` でログイン済み
- Chrome DevTools の「デバイスツールバー」を使用

---

## 3. テスト環境

| 環境 | 確認方法 |
|---|---|
| PC（1920×1080） | Chrome で通常表示 |
| タブレット（768px） | Chrome DevTools → iPad Mini |
| スマートフォン（390px） | Chrome DevTools → iPhone 14 Pro |

---

## 4. テストケース一覧

### 4-1. レスポンシブレイアウト

| No | テスト項目 | 操作手順 | 期待結果 | 優先度 |
|---|---|---|---|---|
| E-01 | ログイン画面（SP） | スマートフォンサイズでログイン画面を表示 | レイアウトが崩れず入力しやすい | 高 |
| E-02 | ホーム画面（SP） | スマートフォンサイズでホーム画面を表示 | ナビゲーションが操作できる | 高 |
| E-03 | 食材追加画面（SP） | スマートフォンサイズで食材追加フォームを表示 | すべての入力欄が表示・操作できる | 高 |
| E-04 | 一覧画面（SP） | スマートフォンサイズで一覧を表示 | 一覧が横スクロールまたは縦並びで表示される | 高 |
| E-05 | タブレット表示 | iPad サイズで全画面を確認 | 2カラムレイアウト等が適切に表示される | 中 |

### 4-2. ナビゲーション

| No | テスト項目 | 操作手順 | 期待結果 | 優先度 |
|---|---|---|---|---|
| E-06 | メニューナビゲーション | ヘッダー・サイドバーのメニューをすべてタップ | 対応するページに遷移する | 高 |
| E-07 | ブラウザバック | 各画面でブラウザの戻るボタンを操作 | 適切な前画面に戻る | 中 |
| E-08 | 直接 URL アクセス | ログイン済み状態で各ページの URL を直接入力 | 正しいページが表示される | 中 |

### 4-3. フォーム・入力操作

| No | テスト項目 | 操作手順 | 期待結果 | 優先度 |
