# テスト計画書：食材管理・音声入力

| 項目 | 内容 |
|---|---|
| 対応 issue | #19 |
| 対象画面 | 食材追加・食材検索・食材編集・一覧 |
| 作成日 | 2026-05-07 |

---

## 1. テスト目的

- 食材の登録・検索・編集・削除が正常に動作すること
- バーコードスキャンで食品情報が取得できること
- 単価（unit_price）が正しく自動計算されること

---

## 2. 前提条件

- `nuu1729` / `Test1111` でログイン済み
- テストデータ（seed-test.sql）が投入済み

---

## 3. テストケース一覧

### 3-1. 食材追加（手動入力）

| No | テスト項目 | 操作手順 | 期待結果 | 優先度 |
|---|---|---|---|---|
| B-01 | 正常な食材追加 | 食材追加画面で名前・価格・数量・単位・購入先をすべて入力して登録 | 食材が登録され一覧に表示される | 高 |
| B-02 | 単価の自動計算 | 価格 300 円・数量 200g で登録 | 単価が `1.5000 円/g` と表示される | 高 |
| B-03 | 必須項目未入力 | 名前を空にして登録 | バリデーションエラーが表示される | 高 |
| B-04 | 数量に 0 を入力 | 数量 0 で登録 | バリデーションエラーが表示される（0除算防止） | 高 |
| B-05 | 価格にマイナスを入力 | 価格 -100 で登録 | バリデーションエラーが表示される | 中 |
| B-06 | ジャンル選択 | ジャンルを「肉類」に設定して登録 | 登録後、一覧でジャンルが正しく表示される | 中 |
