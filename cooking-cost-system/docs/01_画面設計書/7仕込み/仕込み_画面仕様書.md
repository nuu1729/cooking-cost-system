# 仕込み 画面仕様書

## 1. 概要
「仕込み」画面は、登録済みの食材を組み合わせて、新しい仕込み品（半製品）を作成・構成するための画面である。
複数の食材をレシピに合わせて追加することで、その仕込み品の原価を自動算出し、管理することができる。
登録された仕込み品は、以降「食材」と同様に他の仕込みやお品の材料として再利用が可能となる。

## 2. 画面要素
### 2.1 左カラム：食材選択・追加エリア
登録済みの食材を検索し、使用量を入力してリストに追加するエリア。

| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **アイコン** | 画像 | ボックスを模したアイコンを表示。 | `/icons/mid_icon.png` |
| **タイトル** | テキスト | 「仕込み」と表示。 |  |
| **説明文** | テキスト | 「登録した食材を選択して追加してください。」と表示。 |  |
| **商品名（検索）** | テキスト入力 | 入力内容に基づき候補をDBから検索。サジェストを表示し、ユーザーが選択する。 | 初期：空 |
| **量** | 数値入力 | 使用する量を入力。 | 初期：0 |
| **単位** | 表示項目 | 選択された食材に紐づく単位が自動入力される。 | 変更不可 |
| **値段** | 表示項目 | 選択された食材の単価と入力された量から算出結果を表示。 | 初期：0（表示上は ¥ 0） |
| **仕込みに追加** | ボタン | 入力内容を確定し、右側の仕込みリストへ追加する。 |  |

### 2.2 右カラム：構成リスト・合計エリア
追加された食材の一覧と、それらをまとめる仕込み名の設定。

| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **仕込み名（ラベル）** | テキスト | 「仕込み名」と表示。 |  |
| **仕込み名入力** | テキスト入力 | 作成する仕込みの名称（例：ごはん中、合わせ味噌等）を入力。 | 必須。初期：空 |
| **仕込み量** | 数値入力 | この工程で出来上がる総量を入力。 | 必須。単位は「g」固定 |
| **構成リストエリア** | コンテナ | 追加された食材をカード形式でリストアップする。 | 初期：空 |
| **食材カード** | カード | リストに追加された1件の食材情報。 |  |
| **(カード表示項目)** | テキスト | 商品名、量 + 単位、購入先を表示。 |  |
| **編集アイコン** | アイコン | カード内の量を再編集する。 | `/icons/edit_icon.png` |
| **削除アイコン** | アイコン | カードをリストから除外する。 | `/icons/delete_icon.png` |
| **合計金額** | テキスト | リストに含まれる全ての食材の値段の合計を表示。 | 初期：¥ 0、自動計算 |
| **仕込み確定** | ボタン | 全ての入力を確定し、保存処理（確認モーダル）へ進む。 |  |

## 3. 画面イベント
### 3.1 商品検索・サジェスト
- ユーザーが「商品名（検索）」に入力中、バックエンドAPIを呼び出し、一致する食材候補をリスト表示する（Debounce: 300ms）。
- ユーザーが候補から一つを選択した時点で、その食材の「単価情報（価格/量）」、「単位」、「購入先」を取得し、フォームへ反映する。

### 3.2 値段の算出
- 「量」の値が入力・変更された際、選択された食材の単価を用いて値段を動的に算出する。
- **計算式**: `値段 = 選択食材の単価 * 入力された量`
- ※単位は食材登録時のものが自動で適用されるため、画面上での単位変換処理は行わない。

### 3.3 仕込みリストへの追加・編集・削除
- **追加**: 「仕込みに追加」ボタン押下時、バリデーション（商品選択済み、量 > 0）を行い、右側リストへ反映する。反映後、左フォームをクリアする。
- **編集**: 食材カードの「編集アイコン」を押下すると、その食材の情報が左カラムの入力フォームへ戻り、再編集可能な状態になる。
- **削除**: 食材カードの「削除アイコン」を押下すると、即座にリストから除外される。

### 3.4 合計金額の更新
- 右側のリストにデータが追加、編集、削除されたタイミングで、全要素の値段を合算し「合計金額」を更新する。

### 3.5 仕込みの保存（再利用の有効化）
- 「仕込み確定」ボタンが押下されると、以下の処理を行う。
    1. **バリデーション**: 「仕込み名」「仕込み量」が入力されているか、リストに1件以上あるかを確認。
    2. **確認モーダル**: 登録内容の最終確認を表示。
    3. **保存**: 保存成功後、この仕込み品はマスターデータに登録され、検索機能から「食材」のひとつとして呼び出せるようになる。

## 4. API定義

### 4.1 エンドポイント
| エンドポイント名 | メソッド | 説明 |
| :--- | :--- | :--- |
| `/api/ingredients/search` | GET | 商品名のインクリメンタル検索。 |
| `/api/preps` | POST | 仕込みデータの登録。 |

### 4.2 POST `/api/preps` リクエスト構成

```json
{
  "prep_name": "合わせ味噌",
  "yield_amount": 500,
  "yield_unit": "g",
  "total_cost": 1200,
  "items": [
    {
      "ingredient_id": 12,
      "amount": 250,
      "unit": "g",
      "cost": 600
    },
    ...
  ]
}
```

## 5. 画面レイアウト構成
画面は大きく左右の2カラム構成（PC/タブレット前提）で配置される。

```text
+---------------------------------------------------------------+
| [Header] 料理原価計算システム                (NAV) ホーム/検索/etc |
+---------------------------------------------------------------+
|                                                               |
|  [左カラム: 食材選択エリア]           [右カラム: 構成・設定エリア]     |
|                                                               |
|  (i) 仕込み                          +-----------------------+ |
|  * (i) = /icons/mid_icon.png         | 構成リスト(カード一覧)    | |
|                                      | +-------------------+ | |
|  [商品名検索: ________ v]            | | 食材カード 1   [E][D]| | |
|                                      | +-------------------+ | |
|  [量: _______] [単位(g)]             | | 食材カード 2   [E][D]| | |
|                                      | +-------------------+ | |
|  単価目安: ¥ 120                     +-----------------------+ |
|                                                               |
|  [ 仕込みに追加 ]                      [ 下部設定エリア (2カラム) ]   |
|                                      +-----------+-----------+ |
|                                      | (左)      | (右)      | |
|                                      | 仕込み名  | 合計金額  | |
|                                      | [____]    | ¥ 160     | |
|                                      |           |           | |
|                                      | 仕込み量  | [ 確定 ]  | |
|                                      | [____] g  |           | |
|                                      +-----------+-----------+ |
+---------------------------------------------------------------+
```
