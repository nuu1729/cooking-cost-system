# アカウント作成画面 イベント定義

## 1. 画面概要
ユーザーがシステムを新規利用するためにアカウント登録を行う画面。
**JWT (JSON Web Token)** 認証を使用し、登録成功後に自動的にログイン状態とする。

## 2. UI仕様とイベント定義

### 入力フォーム
| 項目 | 必須 | バリデーションルール | イベント (OnChange) |
|---|:---:|---|---|
| **メールアドレス** | 必須 | ・有効なメールアドレス形式であること<br>・既に使用されているメールアドレスではないこと(API側で判定) | ・入力値を状態管理(State)に反映<br>・フォーカスアウト時に形式チェック |
| **パスワード** | 必須 | ・8文字以上<br>・大文字、小文字、数字をすべて含むこと | ・入力値を状態管理(State)に反映 |

### アクション・遷移

#### 1. アカウント作成ボタン (Sign Up Button)
- **配置**: 入力フォームの下部
- **イベント (OnClick)**:
  1. **バリデーションチェック**:
     - メールアドレス・パスワードが入力されているか。
     - バリデーションルール（メール形式、パスワード複雑性）を満たしているか。
     - NGの場合：エラーメッセージを表示し、処理を中断。
  2. **登録リクエスト**:
     - APIへ `POST` リクエスト送信 (ペイロード: `email`, `password`)。
  3. **レスポンス処理**:
     - **成功時 (201 Created)**:
       - レスポンスに含まれるJWTトークンを保存 (LocalStorage/Cookie等)。
       - **「2ホーム画面」へ遷移する。**
     - **失敗時**:
       - エラーメッセージを表示 (例: "このメールアドレスは既に登録されています")。

#### 2. ログイン画面へ戻るリンク (Back to Login)
- **配置**: **アカウント作成ボタンの下**にテキスト「すでにアカウントをお持ちの方はこちら (ログイン)」として配置。
- **イベント (OnClick)**:
  - **「1ログイン画面」へ遷移する。**
  - 現在の `mode` を `signup` から `login` に切り替える等の処理。

#### 3. 将来の拡張予定 (OAuth)
- **Google / Apple での登録**:
  - 将来的に実装予定。ログイン画面と同様に考慮する。

## 3. 詳細ロジック

### バリデーション実装詳細
- **メールアドレス**: 正規表現等を用いた標準的なチェック。
- **パスワード**: 以下の3条件をANDで満たすこと（ログイン画面と統一）。
  - `length >= 8`
  - `/[A-Z]/` (大文字を含む)
  - `/[a-z]/` (小文字を含む)
  - `/[0-9]/` (数字を含む)

### 画面遷移定義
- **Sign Up Success** `->` `2ホーム画面`
- **Back to Login Click** `->` `1ログイン画面`
