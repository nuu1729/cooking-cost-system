# 購入先 画面仕様書

## 1. 概要

購入先（仕入れ先）マスタを管理する画面。食材の購入先を正規化してマスタ管理し、
食材追加・編集画面のセレクトボックスに供給する。
2カラムレイアウトで、左カラムでCRUD操作、右カラムで一覧表示を行う。

**パス**: `/stores`  
**ヘッダー位置**: 「お品」と「販売価格計算」の間

---

## 2. 画面要素

### 左カラム：入力フォーム

| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **購入先名** | テキスト入力 | 購入先の名称を入力（例：コスモス、業務スーパー）。Enterキーで登録/更新を実行。 | 必須 |
| **新規** ボタン | ボタン | フォームをリセットし新規登録モードにする。 | グレー |
| **登録** / **更新** ボタン | ボタン | 新規モード時は「登録」、編集モード時は「更新」を表示。 | 青（#1E90FF）、名前未入力時は非活性 |
| **削除** ボタン | ボタン | 編集モード時のみ表示。選択中の購入先を削除する。 | 赤（#FF4444） |
| **編集中表示** | テキスト | 編集モード時、選択中の購入先名と使用食材数を表示。 | 青背景 |
| **エラーメッセージ** | テキスト | バリデーションエラー・操作失敗時に赤色で表示。 | |
| **成功メッセージ** | テキスト | 操作成功時に緑色で表示。 | |

### 右カラム：一覧

| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **件数バッジ** | テキスト | 登録済み購入先の総件数を表示。 | |
| **ソート説明** | テキスト | 「使用食材数が多い順」と表示。 | |
| **購入先リスト** | リスト | 各購入先行に順位・名称・食材使用件数を表示。クリックで編集モードに切り替わる。 | |

## 3. 画面イベント

### 3.1 ページロード

1. `GET /api/stores` を呼び出し購入先一覧を取得する。
2. 一覧を「食材使用件数（ingredient_count）降順」で表示する。
3. フォームは新規モードの空状態で起動する。

### 3.2 購入先選択（編集モード移行）

1. 右カラムの購入先行をクリックする。
2. 選択した購入先の名前が左フォームの入力欄にセットされる。
3. ボタンが「更新」「削除」表示に切り替わる。
4. 編集中表示に購入先名と使用食材数が表示される。

### 3.3 新規登録

1. 「新規」ボタンを押してフォームをリセットし新規モードにする。
2. 購入先名を入力して「登録」ボタンを押す（またはEnterキー）。
3. `POST /api/stores` を呼び出す。
4. 成功時：成功メッセージを表示し、フォームをリセット、一覧を再取得する。
5. 同名購入先が既存の場合（409）：エラーメッセージを表示する。

### 3.4 購入先更新

1. 右カラムで購入先を選択し編集モードに入る。
2. 左カラムの名前を変更して「更新」ボタンを押す。
3. `PUT /api/stores/{id}` を呼び出す。
4. 成功時：成功メッセージを表示し、一覧を再取得する。
   - バックエンドで `items.store` テキストフィールドも同名で一括更新される。
5. 同名購入先が既存の場合（409）：エラーメッセージを表示する。

### 3.5 購入先削除

1. 右カラムで購入先を選択し編集モードに入る。
2. 「削除」ボタンを押す。
3. `DELETE /api/stores/{id}` を呼び出す。
4. 成功時：成功メッセージを表示し、フォームをリセット、一覧を再取得する。
5. その購入先を使用中の食材が存在する場合（409）：エラーメッセージを表示する（例：「この購入先は 3 件の食材で使用中のため削除できません」）。

---

## 4. ソートロジック

一覧の表示順は **使用食材数（`ingredient_count`）降順** とする。  
`ingredient_count` は `items` テーブルで `item_type = 1` かつ `store_id = stores.id` の件数をカウントしたもの。  
件数が同じ場合の順序は不定。

---

## 5. エラーハンドリング一覧

| 操作 | 条件 | 表示内容 |
| :--- | :--- | :--- |
| 登録・更新 | 名前が空 | 「購入先名を入力してください」 |
| 登録・更新 | 同名購入先が存在（409） | 「その購入先名はすでに登録されています」 |
| 削除 | 食材で使用中（409） | 「この購入先は N 件の食材で使用中のため削除できません」 |
| 全操作 | 通信エラー | 「操作に失敗しました」 |
