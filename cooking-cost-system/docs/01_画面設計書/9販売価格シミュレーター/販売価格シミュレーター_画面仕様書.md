# 販売価格シミュレーター 画面仕様書

## 1. 概要

登録済みの「お品」を選択し、食材原価に基づいて販売価格の適正度を診断・シミュレーションする画面。  
食材価格の変動（インフレ・仕入れコスト上昇）によって原価が押し上げられた際、現在の販売価格が引き続き健全かを即座に確認できる。

---

## 2. 採用する経営理論・指標

### 2.1 三分法（飲食業の経験則）

日本の飲食業界で広く用いられる原価管理の基準。

| 区分 | 目安 | 内容 |
|---|---|---|
| 食材費（F: Food Cost） | 30% | 原材料費 |
| 人件費（L: Labor Cost） | 30% | スタッフ給与・社保等 |
| 諸経費（E: Expense） | 30% | 家賃・光熱費・消耗品等 |
| 利益 | 10% | 税引前純利益の目安 |

> 本システムは食材費のみ把握可能なため、三分法を基準として「食材費が30%以内」を健全ラインとする。

### 2.2 原価率（Food Cost Ratio）

```
原価率 = 食材原価 ÷ 販売価格 × 100  [%]
```

- 飲食業界の一般的な目標値：**25〜35%**
- 業態別目安：カフェ 25〜30% / 定食・ファミレス 30〜35% / 高級料理 28〜38%

### 2.3 粗利益・粗利益率（Gross Profit / Gross Profit Margin）

```
粗利益   = 販売価格 - 食材原価
粗利益率 = 粗利益 ÷ 販売価格 × 100  [%]
```

粗利益から人件費・諸経費を回収するため、粗利益率は **65%以上** が健全ライン（原価率30%のとき粗利益率70%）。

### 2.4 FL比率（Food & Labor Cost Ratio）

```
FL比率 = (食材費 + 人件費) ÷ 売上 × 100  [%]
```

- **60%以下**：健全（業界基準）
- 60〜65%：注意域
- 65%超：経営リスク大
- 本画面では人件費を売上の30%と推定して表示する。

### 2.5 推奨販売価格の計算

目標原価率（ユーザーが設定可能）を基に逆算する。

```
推奨販売価格 = 食材原価 ÷ 目標原価率
例：原価 300円 ÷ 0.30 = 推奨価格 1,000円
```

### 2.6 コスト変動耐性（Price-Cost Sensitivity）

仕入れ価格が上昇した場合、現在の販売価格を維持したままどの程度耐えられるかを計算。

```
コスト増加後の原価率 = (現在の食材原価 × (1 + 上昇率)) ÷ 販売価格
```

## 3. 判定基準

| マーク | 判定 | 原価率 | 意味 |
|---|---|---|---|
| ◎ | **最適** | < 30% | 三分法の目標以内。十分な利益確保が期待できる。 |
| ○ | **適正** | 30〜35% | 業界標準範囲内。概ね健全。 |
| △ | **要注意** | 35〜50% | 人件費・経費まで含めると収益が厳しくなる可能性あり。 |
| ✗ | **赤字リスク** | 50%以上 | 食材費だけで売上の半分を超える。価格改定を強く推奨。 |

## 4. 画面要素

### 4.1 左カラム：入力エリア

| 要素名 | 種類 | 説明 | 備考 |
|---|---|---|---|
| **ページタイトル** | 表示 | 「販売価格シミュレーター」 | アイコン付き |
| **お品選択** | セレクトボックス | 登録済みのお品一覧から選択 | 選択するとデータが自動セット |
| **食材原価** | 表示（読み取り専用） | 選択したお品の登録済み食材原価 | 自動セット、変更不可 |
| **販売価格** | 数値入力 | シミュレーション対象の販売価格 | お品の登録価格を初期値として表示。手入力で変更可能 |
| **目標原価率** | スライダー | 20〜50%の範囲で設定 | 初期値：30%（三分法基準） |
| **推奨販売価格** | 表示 | 目標原価率に基づく逆算価格 | スライダー連動でリアルタイム更新 |

### 4.2 右カラム：シミュレーション結果

| 要素名 | 種類 | 説明 |
|---|---|---|
| **判定バッジ** | 表示 | ◎/○/△/✗ と判定名を大きく表示 |
| **主要指標カード** | カード群 | 現在の原価率 / 粗利益 / 粗利益率 / 推定FL比率 を表示 |
| **三分法内訳** | 構成表示 | 販売価格を三分法で分解したイメージを表示 |
| **価格帯シミュレーション** | テーブル | 現在価格±30%の範囲で複数価格帯の指標を一覧表示 |
| **コスト上昇シミュレーション** | リスト | 食材原価が+5/10/15/20/30%上昇した場合の原価率・判定変化を表示 |

---

## 5. 画面イベント

### 5.1 お品選択
- セレクトボックスを変更すると、食材原価・販売価格が自動セットされる。
- 右カラムの全指標がリアルタイムで更新される。

### 5.2 販売価格の手入力
- 数値を入力するたびに全指標がリアルタイム更新される。
- 非数値・0以下は入力を無視し、エラー状態を表示する。

### 5.3 目標原価率スライダー
- スライダーを動かすと推奨販売価格がリアルタイム更新される。
- 判定基準は固定（スライダーの変動による判定は変わらない）。

## 7. 注意事項・制約

- 人件費・諸経費はシステム上保持していないため、「三分法（各30%）」を使った**推定値**として表示する。
- 推奨販売価格は10円単位で丸める（切り上げ）。
- 価格帯シミュレーションの各行は現在価格の70〜130%を等間隔で表示する。
- データ保存は行わない（シミュレーション専用画面）。
