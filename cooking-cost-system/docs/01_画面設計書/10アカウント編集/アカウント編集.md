# アカウント編集画面 画面設計書

## 1. 画面概要

ログイン中のユーザーが自身のアカウント情報を閲覧・編集する画面。  
ヘッダー左端の**アカウントアイコン**を押下することで遷移する。

| 項目 | 内容 |
|---|---|
| 画面名 | アカウント編集 |
| URL パス | `/account` |
| 認証要否 | **必須**（未ログイン時はログイン画面へリダイレクト） |
| レイアウト | 共通レイアウト（Header + main 領域） |

---

## 2. 遷移条件（ヘッダーアイコン押下時）

```
アカウントアイコン押下
     │
     ├─ ログイン済み（localStorage に authToken が存在）
     │       └─→ アカウント編集画面 (/account) へ遷移
     │
     └─ 未ログイン（authToken が存在しない）
             └─→ ログイン画面 (/login) へリダイレクト
```

> **実装メモ**  
> `localStorage.getItem('authToken')` の有無で判定する。  
> またはルートガード（Protected Route）側で共通制御してもよい。

---

## 3. 初期表示

画面マウント時に **`GET /api/auth/me`** を呼び出してログイン中ユーザーの情報を取得し、各フィールドの初期値として表示する。

### 取得エンドポイント

| 項目 | 内容 |
|---|---|
| メソッド | `GET` |
| パス | `/api/auth/me` |
| 認証ヘッダー | `Authorization: Bearer {authToken}` |

### レスポンス（成功時 `200 OK`）

```json
{
  "success": true,
  "data": {
    "id": 1,
    "username": "山田 太郎",
    "email": "taro@example.com",
    "role": "user",
    "is_active": true,
    "last_login": "2026-03-20T10:00:00Z",
    "created_at": "2026-01-01T00:00:00Z",
    "updated_at": "2026-03-20T10:00:00Z"
  },
  "timestamp": "2026-03-20T13:00:00Z"
}
```

## 5. 入力フォーム仕様

### セクション①：アカウントアイコン

| 項目 | 内容 |
|---|---|
| 必須 | 任意 |
| アップロード方法 | クリック（ファイルダイアログ） / ドラッグ＆ドロップ |
| 許可形式 | PNG / JPG / GIF / WebP |
| 最大ファイルサイズ | 5 MB |
| 推奨サイズ | 正方形（例：200×200px、400×400px） |
| 表示形状 | 内接円形（`border-radius: 50%` + `object-fit: cover`） |
| ヘッダー表示サイズ | 直径 40px |
| プレビューサイズ | 直径 100px（この画面内のみ） |
| アイコン未設定時 | 人型シルエットSVGを表示 |
| 削除ボタン | アイコン登録済みのときのみ表示。押下でプレビューをリセット |

> **技術メモ**  
> 現フェーズではアイコン画像は `localStorage` に base64 で保存する。  
> 将来的にはバックエンドへのアップロード API（`POST /api/upload/avatar` 等）に切り替える予定。

---

### セクション②：基本情報

#### アカウント名（任意）

| 項目 | 内容 |
|---|---|
| フィールド名 | `username` |
| 種別 | テキスト入力 |
| 必須 | いいえ |
| 最大文字数 | 50文字 |
| バリデーション | なし（任意入力のため） |
| 初期値 | DBから取得した `username` |
| プレースホルダー | `例：山田 太郎` |

#### メールアドレス

| 項目 | 内容 |
|---|---|
| フィールド名 | `email` |
| 種別 | メール入力 (`type="email"`) |
| 必須 | はい |
| バリデーション | ・有効なメールアドレス形式であること（`/.+@.+\..+/` 等）<br>・**変更がある場合のみ**バリデーションを実施<br>・変更がない場合はそのまま送信を許可 |
| 初期値 | DBから取得した `email` |
| プレースホルダー | `例：taro@example.com` |
| エラーメッセージ | `有効なメールアドレスを入力してください` |

---

### セクション③：パスワード変更

パスワードを変更する場合のみ入力する。**3フィールドすべてが空のときはパスワード変更APIを呼び出さない。**  
1フィールドでも入力値がある場合は3フィールドすべてのバリデーションを実施する。

#### 現在のパスワード

| 項目 | 内容 |
|---|---|
| フィールド名 | `currentPassword` |
| 種別 | パスワード入力（トグル表示可） |
| 必須 | パスワード変更時のみ必須 |
| バリデーション | 入力必須（パスワード変更時） |
| 目玉ボタン | マウス押下中のみ表示（`onMouseDown` / `onMouseUp`） |

#### 新しいパスワード

| 項目 | 内容 |
|---|---|
| フィールド名 | `newPassword` |
| 種別 | パスワード入力（トグル表示可） |
| 必須 | パスワード変更時のみ必須 |
| バリデーション | アカウント作成時と同じ条件（下記参照） |
| プレースホルダー | `8文字以上（大文字・小文字・数字を含む）` |

**パスワード複雑性ルール（アカウント作成時と統一）**

| ルール | 内容 |
|---|---|
| 文字数 | 8文字以上 |
| 大文字 | 1文字以上含むこと (`/[A-Z]/`) |
| 小文字 | 1文字以上含むこと (`/[a-z]/`) |
| 数字 | 1文字以上含むこと (`/[0-9]/`) |

#### 新しいパスワード（確認）

| 項目 | 内容 |
|---|---|
| フィールド名 | `confirmNewPassword` |
| 種別 | パスワード入力（トグル表示可） |
| 必須 | パスワード変更時のみ必須 |
| バリデーション | `newPassword` と完全一致すること |
| エラーメッセージ | `パスワードが一致しません` |

## 6. アクション・遷移定義

### 保存するボタン

- **配置**: 画面下部
- **イベント (OnClick)**:

```
① フォームバリデーション実行
   ├─ NG → エラーメッセージを表示し処理を中断
   └─ OK → 以下の API を順次呼び出す

② プロフィール更新 API（基本情報に変更がある場合）
   PUT /api/auth/profile
   Body: { username, email }

③ パスワード更新 API（パスワードフィールドに入力がある場合）
   PUT /api/auth/password
   Body: { currentPassword, newPassword }

④ アカウントアイコン保存（変更がある場合）
   ※現フェーズ: localStorageに保存
   ※将来: POST /api/upload/avatar

⑤ 全 API 成功時
   → ホーム画面 (/) へ遷移
   → ヘッダーのアカウントアイコンを最新状態に更新

⑥ API 失敗時
   → エラーメッセージをインラインで表示
   → 画面遷移しない
```

## 7. API 仕様

### プロフィール更新

| 項目 | 内容 |
|---|---|
| メソッド | `PUT` |
| パス | `/api/auth/profile` |
| 認証ヘッダー | `Authorization: Bearer {authToken}` |

**リクエストボディ**

```json
{
  "username": "山田 太郎",
  "email": "new@example.com"
}
```

**レスポンス（成功時 `200 OK`）**

```json
{
  "success": true,
  "data": {
    "id": 1,
    "username": "山田 太郎",
    "email": "new@example.com"
  },
  "message": "プロフィールを更新しました",
  "timestamp": "2026-03-20T13:00:00Z"
}
```

**エラーレスポンス例**

| ステータスコード | 内容 | 表示メッセージ |
|---|---|---|
| `400` | メール形式不正 | `有効なメールアドレスを入力してください` |
| `409` | メールアドレス重複 | `このメールアドレスはすでに使用されています` |
| `401` | 未認証 | ログイン画面へリダイレクト |

---

### パスワード更新

| 項目 | 内容 |
|---|---|
| メソッド | `PUT` |
| パス | `/api/auth/password` |
| 認証ヘッダー | `Authorization: Bearer {authToken}` |

**リクエストボディ**

```json
{
  "currentPassword": "OldPass123",
  "newPassword": "NewPass456"
}
```

**レスポンス（成功時 `200 OK`）**

```json
{
  "success": true,
  "message": "パスワードを更新しました",
  "timestamp": "2026-03-20T13:00:00Z"
}
```

**エラーレスポンス例**

| ステータスコード | 内容 | 表示メッセージ |
|---|---|---|
| `400` | パスワード複雑性不足 | `パスワードは8文字以上、大文字・小文字・数字を含む必要があります` |
| `401` | 現在のパスワード誤り | `現在のパスワードが正しくありません` |

## 8. バリデーション一覧

| フィールド | タイミング | ルール |
|---|---|---|
| メールアドレス | フォーカスアウト時・保存ボタン押下時 | 有効なメールアドレス形式（変更がある場合のみ） |
| 現在のパスワード | 保存ボタン押下時 | パスワード変更時は入力必須 |
| 新しいパスワード | フォーカスアウト時・保存ボタン押下時 | 8文字以上、大文字・小文字・数字を含む |
| 新しいパスワード（確認） | フォーカスアウト時・保存ボタン押下時 | 新しいパスワードと一致すること |

## 9. 画面遷移定義

```
[ヘッダー：アカウントアイコン押下]
     ├─ 未ログイン  →  /login（ログイン画面）
     └─ ログイン済み →  /account（本画面）

[保存するボタン押下]
     ├─ バリデーションNG  →  本画面にとどまる（エラー表示）
     ├─ API成功           →  / （ホーム画面）
     └─ API失敗           →  本画面にとどまる（エラー表示）
```
