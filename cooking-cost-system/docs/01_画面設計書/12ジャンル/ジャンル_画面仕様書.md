# ジャンル管理画面 仕様書

## 1. 基本情報

| 項目 | 内容 |
|------|------|
| 画面ID | SCR-012 |
| 画面名 | ジャンル管理 |
| URL | /genres |
| 対象ユーザー | ログイン済みユーザー |
| 目的 | 食材ジャンルのマスタ管理（登録・編集・削除） |

## 3. 機能仕様

### 3.1 ジャンル登録

- ジャンル名（テキストボックス）を入力して「登録する」ボタンを押す
- 同名のジャンルがすでに存在する場合はエラートースト表示（409）
- 登録成功後：一覧を再読み込みし、入力フィールドをクリア

### 3.2 ジャンル編集

- 一覧の行をクリックすると、そのジャンルが選択状態になる
- 選択されたジャンルの名前がテキストボックスに入力される
- 「更新する」ボタンで名前を変更
- 更新時、そのジャンル名を使用している全食材の `genre` テキスト列も同期更新する

### 3.3 ジャンル削除

- 選択中のジャンルに紐づく食材が 0 件の場合のみ「削除する」ボタンが活性
- 削除確認モーダルを表示してから削除実行
- 食材が 1 件以上ある場合はボタンが非活性（409 保護）

### 3.4 一覧表示

- 食材使用数の多い順に表示
- 使用中のジャンルには「使用中」バッジを表示
- 選択中のジャンルには「選択中」バッジを表示

## 4. バリデーション

| フィールド | ルール |
|-----------|--------|
| ジャンル名 | 必須、最大50文字 |
| 重複チェック | 同ユーザー内で同名不可（大文字小文字区別） |

## 5. API連携

| 操作 | メソッド | エンドポイント |
|------|---------|--------------|
| 一覧取得 | GET | /api/genres |
| 登録 | POST | /api/genres |
| 更新 | PUT | /api/genres/{id} |
| 削除 | DELETE | /api/genres/{id} |
