# 食材検索 画面イベント設計

## 1. 概要
登録済みの食材を様々な条件で検索し、最適な購入先（最安値など）を比較・特定するための画面である。
単位の異なる食材（例：個とg）を同一条件で比較するための計算機能を備える。

## 2. 画面要素
### ヘッダー・検索入力
- **商品名入力**: 食材の名前で絞り込み。リアルタイム検索に対応。
- **音声入力ボタン**: 音声で食材名を指定可能。

### 検索条件（プレミアム検索パネル）
| 要素 | 説明 |
| :--- | :--- |
| **最安値** | 単価（100g/ml/単位あたり）が安い順にソートし、No.1を強調表示する。 |
| **量を揃える** | 比較しやすいよう、指定された100単位（100g等）あたりの参考価格を表示する。 |
| **1個を別の単位に揃える** | 「個」で登録された食材を「g」に換算して比較する。換算率は入力可能。 |

### 候補リスト（カード形式）
- **候補 1〜N**: 検索結果をカード形式で羅列。
- **ハイライト**: 最安値の候補には「Best Value」バッジとオレンジの枠線を表示。

## 3. 画面イベント
### 3.1 リアルタイム検索
- 商品名が入力されるたびにバックエンドAPIを呼び出し、結果を即座に反映。

### 3.2 単位換算比較
- 「1個を別の単位に揃える」がONの場合、換算率（例：1個=200g）を用いて計算された単価で最安値を判定する。

### 3.3 音声認識
- 音声から食材名を抽出し、検索窓に自動セットして検索を実行する。
