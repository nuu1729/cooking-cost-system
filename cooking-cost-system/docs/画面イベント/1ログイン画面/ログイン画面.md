# ログイン画面 イベント定義

## 1. 画面概要
ユーザーがシステムを利用するために認証を行う画面。
JWT認証を採用し、有効なクレデンシャル（メールアドレス・パスワード）を入力することでアクセストークンを取得する。

## 2. UI仕様

### 入力フィールド
| 項目名 | 必須 | 制約・バリデーション | 備考 |
| :--- | :---: | :--- | :--- |
| **メールアドレス** | 必須 | ・メールアドレス形式であること<br>・未入力不可 | |
| **パスワード** | 必須 | ・8文字以上<br>・大文字、小文字、数字をそれぞれ1文字以上含む<br>・未入力不可 | マスキング表示（表示/非表示切替機能推奨） |

### アクション
| ボタン/リンク | 動作概要 | 遷移先 |
| :--- | :--- | :--- |
| **ログインボタン** | 入力内容を検証し、APIへログインリクエストを送信する | 成功時：ホーム画面 (`/`)<br>失敗時：エラーメッセージ表示 |
| **アカウント作成リンク** | 新規登録モードへ切り替える（将来実装） | 同画面内でモード切替 (予定) |
| **パスワード忘れ** | パスワードリセット手続きへ（将来実装） | - |

## 3. イベント詳細

### 3.1 初期表示 (OnLoad)
- **処理**: 
  - すでに有効なJWTを持っている場合は、自動的にホーム画面へリダイレクトする（任意実装）。
  - 入力フィールドは空の状態（またはブラウザのオートフィル）。
  - フォーカスは「メールアドレス」に当てる。

### 3.2 入力変更 (OnChange)
- **対象**: メールアドレス、パスワード
- **処理**:
  - 入力値をStateに反映する。
  - リアルタイムバリデーションは任意（UX向上のため、フォーカスアウト時または入力中に形式チェックを行う）。

### 3.3 ログイン実行 (OnSubmit / OnClick)
- **トリガー**: ログインボタン押下 または パスワード欄でEnterキー押下
- **前処理 (クライアントバリデーション)**:
  1. **必須チェック**: メール、パスワードが入力されているか。
  2. **形式チェック**:
     - メールアドレスが正しい形式か。
     - パスワードがポリシー（8文字以上、大文字・小文字・数字含む）を満たしているか。
  - **NGの場合**: 該当フィールド下にエラーメッセージを表示し、API通信は行わない。
- **API通信**:
  - メソッド: `POST`
  - エンドポイント: `/api/auth/login` (仮)
  - ペイロード: `{ "email": "...", "password": "..." }`
- **後処理**:
  - **成功 (200 OK)**:
    - レスポンスからJWTを取得。
    - ローカルストレージ (またはCookie) にトークンを保存。
    - ホーム画面へ遷移 (`navigate('/')`)。
  - **失敗 (401 Unauthorized 等)**:
    - 「メールアドレスまたはパスワードが間違っています」等のエラーを表示。
    - パスワード欄をクリアする。

### 3.4 モード切替 (将来実装)
- **対象**: 「アカウントを作成」リンク
- **処理**:
  - 現在の `mode` State を `login` から `signup` に変更する。
  - タイトルボタンの文言を「アカウント作成」に変更する。

## 4. エラーハンドリング
- **ネットワークエラー**: サーバに接続できない場合、「通信エラーが発生しました」と表示。
- **バリデーションエラー**: 各入力欄の下に赤字で具体的理由を表示。
