# 食材追加 画面イベント設計

## 1. 概要
本画面は、新しく購入した食材や調味料をシステムに登録するための画面である。
左側にフォーム、右側にアクションボタン（音声入力等）を配置した2カラム構成のプレミアムデザインを採用する。

## 2. 画面要素
### 左カラム：入力フォーム
| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **商品名** | テキスト入力 | 食材の名称を入力（例：トマト）。 | 必須 |
| **価格** | 数値入力 | 購入金額を入力（¥プレフィックス表示）。 | 必須、最小値: 0 |
| **量** | 数値入力 | 購入した「量」を数値で入力。 | 必須、最小値: 0、小数可 |
| **単位** | セレクト入力 | 「g」「ml」「個」から選択。 | 必須 |
| **購入先** | テキスト入力 | 仕入れ先やスーパー名を入力（例：コスモス）。 | 必須 |

### 右カラム：アクション
| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **音声で入力** | ボタン | 音声認識を開始し、各項目を自動入力する。 | パープル/ピンクのグラデーション |
| **食材を登録** | ボタン | 入力内容を確定し、確認画面を表示する。 | 緑色（#53B69B）、「＋」アイコン付き |

### 確認モーダル
| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **登録内容一覧** | リスト | 入力されたすべての内容のサマリーを表示。 | |
| **登録を確定する** | ボタン | サーバーへの送信を実行する。 | Enterキーでも実行可能 |
| **戻る** | ボタン | 入力画面に戻る。 | ESCキーでも実行可能 |

## 3. 画面イベント
### 3.1 ページロード
- フォーム全体がフェードイン。項目は初期化済み。

### 3.2 バリデーションエラー（入力不足）
- 「食材を登録」ボタン押下時にチェックを実施。
- 未入力の項目がある場合：
    - 入力フィールドの枠線を **赤色（border-red-300）** に変更。
    - 背景を薄い赤（bg-red-50）に変更。
    - ラベルの右横に「入力してください」というメッセージを表示。

### 3.3 登録内容の確認（モーダル表示）
- 入力エラーがない場合、確認用のモーダルを中央に表示（背景はブラー付きの暗転）。
- **キー操作サポート**:
    - **Enterキー**: 「登録を確定する」ボタンのクリックと同等の動作。
    - **BackSpace/ESCキー**: 確認画面を閉じ、入力画面に戻る。

### 3.4 登録実行
- 確認画面で「登録を確定する」をクリックした際：
    1. ボタンをローディング状態（登録中...）にする。
    2. バックエンドAPIへデータを送信。
    3. 成功時：完了アラートを表示し、入力値をクリアしてモーダルを閉じる。
