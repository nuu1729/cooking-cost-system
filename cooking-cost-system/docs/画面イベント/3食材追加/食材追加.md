# 食材追加 画面イベント設計

## 1. 概要
本画面は、新しく購入した食材や調味料をシステムに登録するための画面である。
左側にフォーム、右側にアクションボタン（音声入力等）を配置した2カラム構成のプレミアムデザインを採用する。

## 2. 画面要素
### 左カラム：入力フォーム
| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **商品名** | テキスト入力 | 食材の名称を入力（例：トマト）。 | 必須 |
| **価格** | 数値入力 | 購入金額を入力（¥プレフィックス表示）。 | 必須、最小値: 0 |
| **量** | 数値入力 | 購入した「量」を数値で入力。 | 必須、最小値: 0、小数可 |
| **単位** | セレクト入力 | 「g」「ml」「個」から選択。 | 必須 |
| **購入先** | テキスト入力 | 仕入れ先やスーパー名を入力（例：コスモス）。 | 必須 |

### 右カラム：アクション
| 要素名 | 種類 | 説明 | 備考 |
| :--- | :--- | :--- | :--- |
| **音声で入力** | ボタン | 音声認識を開始し、各項目を自動入力する。 | パープル/ピンクのグラデーション |
| **食材を登録** | ボタン | 入力内容を確定し、サーバーへ送信。 | 緑色（#53B69B）、「＋」アイコン付き |

## 3. 画面イベント
### 3.1 ページロード
- フォームとアクションボタンが統合されたレイアウトを表示。
- 各項目は空の状態で初期化（単位のみデフォルト「g」）。

### 3.2 入力バリデーション
- 全項目が必須（HTML5 standard validation）。
- 価格・量は、マイナスの値を制限。

### 3.3 音声入力（将来拡張）
- 「音声で入力」ボタンクリックにより音声認識モードへ。
- 認識結果に基づき、「商品名」「価格」「量」などの各フィールドを自動的に埋める。

### 3.4 登録実行
- 「食材を登録」ボタンをクリックした際、以下の処理を行う。
  1. ボタンを無効化し、ローディング状態を表示。
  2. 入力データを収集し、バックエンドAPIへ送信。
  3. 成功時、トースト通知等で完了を知らせ、入力値をリセットする。
