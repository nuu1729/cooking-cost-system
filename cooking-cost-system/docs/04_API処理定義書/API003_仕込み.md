## 3. 各エンドポイント処理詳細

### 3-1. GET /api/preps — 仕込み一覧取得

#### クエリパラメータ

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| `name` | string | — | 仕込み名（部分一致） |
| `genre` | string | — | ジャンル（完全一致） |
| `sortBy` | string | `name` | `name` / `price` / `created_at` |
| `sortOrder` | string | `ASC` | `ASC` / `DESC` |
| `limit` | integer | 20 | 最大 100 |
| `offset` | integer | 0 | — |

#### 処理フロー

1. JWT を検証する
2. クエリパラメータをバリデーションする（`sortBy` 許可リスト確認）
3. `item_type = 2` で絞り込み、`items` テーブルを検索する
4. `{ success, data, count, timestamp }` を返す（`ingredients` リストは含めない）

#### DB操作

```sql
SELECT id, name, item_type, store, price, quantity, unit, unit_price,
       genre, description, created_at, updated_at
FROM items
WHERE item_type = 2
  AND (:name IS NULL OR name LIKE CONCAT('%', :name, '%'))
  AND (:genre IS NULL OR genre = :genre)
ORDER BY :sortBy :sortOrder
LIMIT :limit OFFSET :offset;
```

### 3-2. POST /api/preps — 仕込み登録（確定）

#### バリデーションルール

| フィールド | 必須 | ルール |
|---|:---:|---|
| `name` | ✓ | 1〜255文字 |
| `quantity` | ✓ | 数値、`> 0`（仕上がり量） |
| `unit` | ✓ | 1〜20文字 |
| `genre` | — | 50文字以内 |
| `description` | — | 任意文字列 |
| `items` | ✓ | 配列、1件以上 |
| `items[].ingredient_id` | ✓ | 正の整数、`items` テーブルに存在する食材（`item_type = 1`） |
| `items[].amount` | ✓ | 数値、`> 0` |

#### 処理フロー

1. JWT を検証する
2. リクエストボディをバリデーションする
3. `items[]` の全 `ingredient_id` が `items (item_type = 1)` に存在するか確認する
4. 各食材の `unit_price` を取得し、`cost = unit_price × amount` を計算する
5. `price（総コスト）= 全 cost の合計` を算出する
6. `unit_price = price ÷ quantity` を計算する（仕込み品自体の単価）
7. トランザクション内で以下を実行する：
   - `items` に `item_type = 2` でレコードを `INSERT` する
   - `item_relations` に各食材 `(parent_id=仕込みid, child_id=ingredient_id, amount, cost)` を `INSERT` する
8. 作成された仕込み品レコードを返す

#### DB操作

```sql
-- トランザクション開始
BEGIN;

-- 仕込み品レコード登録
INSERT INTO items (name, item_type, store, price, quantity, unit, unit_price, genre, description, created_at, updated_at)
VALUES (:name, 2, '自家製', :total_cost, :quantity, :unit, :unit_price, :genre, :description, NOW(), NOW());

-- 構成食材の関連登録（ループ）
INSERT INTO item_relations (parent_id, child_id, amount, cost, created_at)
VALUES (:prep_id, :ingredient_id, :amount, :cost, NOW());

COMMIT;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| バリデーション失敗 | 400 | `VALIDATION_ERROR` |
| 存在しない `ingredient_id` | 400 | `VALIDATION_ERROR` |
| 認証失敗 | 401 | `UNAUTHORIZED` |

### 3-3. GET /api/preps/check-name — 仕込み名重複チェック

#### クエリパラメータ

| パラメータ | 必須 | 説明 |
|---|:---:|---|
| `name` | ✓ | 確認する仕込み名 |

#### 処理フロー

1. JWT を検証する
2. `SELECT id FROM items WHERE item_type = 2 AND name = :name` を実行する
3. 存在する場合は `{ exists: true }`、存在しない場合は `{ exists: false }` を返す

> フロントエンドが「仕込み確定」ボタン押下前または仕込み名入力後にリアルタイムチェックするために使用する。

---

### 3-4. GET /api/preps/{id} — 仕込み詳細取得

#### 処理フロー

1. JWT を検証する
2. `SELECT * FROM items WHERE id = :id AND item_type = 2` で仕込み品を取得する（404 チェック）
3. `item_relations` を JOIN して構成食材リストを取得する
4. 仕込み品情報 + `ingredients[]` 配列を返す

#### DB操作

```sql
-- 仕込み品基本情報
SELECT id, name, item_type, store, price, quantity, unit, unit_price,
       genre, description, created_at, updated_at
FROM items
WHERE id = :id AND item_type = 2;

-- 構成食材一覧
SELECT
  ir.child_id AS ingredient_id,
  i.name AS ingredient_name,
  i.unit AS ingredient_unit,
  i.genre AS ingredient_genre,
  ir.amount,
  ir.cost
FROM item_relations ir
JOIN items i ON ir.child_id = i.id
WHERE ir.parent_id = :id
ORDER BY ir.id ASC;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |

### 3-5. DELETE /api/preps/{id} — 仕込み削除

#### 処理フロー

1. JWT を検証する
2. `SELECT id FROM items WHERE id = :id AND item_type = 2` で存在確認する（404 チェック）
3. `item_relations` テーブルで `child_id = :id` かつ親が `item_type = 3`（お品）のレコードを確認する
4. お品で使用中の場合は 409 を返す（使用中のお品名をメッセージに含める）
5. トランザクション内で以下を実行する：
   - `item_relations WHERE parent_id = :id` を `DELETE`（構成食材の関連削除）
   - `items WHERE id = :id AND item_type = 2` を `DELETE`
6. `{ success: true, message: "仕込みを削除しました" }` を返す

> `item_relations` への CASCADE DELETE が設定されている場合は手動削除不要。DB設計に合わせて実装すること。

#### DB操作

```sql
-- お品での使用チェック
SELECT ir.parent_id, i.name AS parent_name
FROM item_relations ir
JOIN items i ON ir.parent_id = i.id
WHERE ir.child_id = :id AND i.item_type = 3
LIMIT 1;

-- 構成食材の関連削除（CASCADE未設定の場合）
DELETE FROM item_relations WHERE parent_id = :id;

-- 仕込み品削除
DELETE FROM items WHERE id = :id AND item_type = 2;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |
| お品で使用中 | 409 | `CONFLICT` |

## 4. セキュリティ・設計注意事項

- 仕込み品登録は必ずトランザクションで行い、`items` と `item_relations` の整合性を保証する
- `ingredient_id` の存在チェックはアプリ層で実施する（外部キー制約がない場合の対策）
- `item_type = 2` の絞り込みを全クエリに必ず付ける
- コスト計算（`unit_price × amount`）はサーバー側で行い、クライアント送信値を使わない
