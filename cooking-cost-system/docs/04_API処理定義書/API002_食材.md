## 3. 各エンドポイント処理詳細

### 3-1. GET /api/ingredients — 食材一覧取得

#### クエリパラメータ

| パラメータ | 型 | デフォルト | 説明 |
|---|---|---|---|
| `name` | string | — | 部分一致フィルター |
| `store` | string | — | 部分一致フィルター |
| `genre` | string | — | 完全一致フィルター |
| `minPrice` | number | — | `price >= minPrice` |
| `maxPrice` | number | — | `price <= maxPrice` |
| `sortBy` | string | `name` | `name` / `price` / `unit_price` / `store` / `created_at` / `updated_at` |
| `sortOrder` | string | `ASC` | `ASC` / `DESC` |
| `limit` | integer | 20 | 最大 100 |
| `offset` | integer | 0 | — |

#### 処理フロー

1. JWT を検証する
2. クエリパラメータを受け取りバリデーションする（`limit` 上限チェック等）
3. `sortBy` が許可リストに含まれるか確認する（SQLインジェクション対策）
4. 動的WHERE句を構築し `SELECT` を実行する
5. `{ success, data, count, timestamp }` を返す

#### DB操作

```sql
SELECT id, name, item_type, store, price, quantity, unit, unit_price,
       genre, description, created_at, updated_at
FROM items
WHERE item_type = 1
  AND (:name IS NULL OR name LIKE CONCAT('%', :name, '%'))
  AND (:store IS NULL OR store LIKE CONCAT('%', :store, '%'))
  AND (:genre IS NULL OR genre = :genre)
  AND (:minPrice IS NULL OR price >= :minPrice)
  AND (:maxPrice IS NULL OR price <= :maxPrice)
ORDER BY :sortBy :sortOrder    -- アプリ層でカラム名を許可リスト検証後に埋め込む
LIMIT :limit OFFSET :offset;
```

### 3-2. POST /api/ingredients — 食材登録

#### バリデーションルール

| フィールド | 必須 | ルール |
|---|:---:|---|
| `name` | ✓ | 1〜255文字 |
| `store_id` | ✓ | 整数、ログインユーザーが所有する `stores.id` |
| `price` | ✓ | 数値、`>= 0` |
| `quantity` | ✓ | 数値、`> 0` |
| `unit` | ✓ | 1〜20文字 |
| `genre` | — | 50文字以内、`meat`/`vegetable`/`seasoning`/`sauce`/`frozen`/`drink`/`other` のいずれか |
| `description` | — | 任意文字列 |

#### 処理フロー

1. JWT を検証する
2. リクエストボディをバリデーションする
3. `stores` テーブルで `store_id` の存在確認（`user_id` も照合）を行う
4. `stores.name` を取得して `items.store` テキストに設定する（後方互換・表示用）
5. `unit_price = price ÷ quantity` を計算する（小数点4桁に丸める）
6. `items` テーブルに `item_type = 1` で `INSERT` する（`store_id` と `store` を両方格納）
7. 挿入されたレコードを返す

#### DB操作

```sql
-- 購入先の存在確認
SELECT id, name FROM stores WHERE id = :store_id AND user_id = :user_id;

-- 食材登録
INSERT INTO items (name, item_type, store, store_id, price, quantity, unit, unit_price, genre, description, user_id, created_at, updated_at)
VALUES (:name, 1, :store_name, :store_id, :price, :quantity, :unit, :unit_price, :genre, :description, :user_id, NOW(), NOW());
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| バリデーション失敗 | 400 | `VALIDATION_ERROR` |
| `store_id` が存在しない | 400 | `VALIDATION_ERROR` |
| 認証失敗 | 401 | `UNAUTHORIZED` |

### 3-3. GET /api/ingredients/search — インクリメンタル検索

#### クエリパラメータ

| パラメータ | 必須 | 説明 |
|---|:---:|---|
| `q` | ✓ | 検索キーワード（1文字以上） |

#### 処理フロー

1. JWT を検証する
2. `q` パラメータのワイルドカード文字（`%`、`_`、`\`）をエスケープする
3. `item_type = 1` かつ `name LIKE '%{q}%'` で検索する
4. 結果を最大20件返す（固定）
5. フロントエンドは Debounce（300ms）制御で呼び出すこと

#### DB操作

```sql
SELECT id, name, store, price, quantity, unit, unit_price, genre
FROM items
WHERE item_type = 1
  AND name LIKE CONCAT('%', :q_escaped, '%')
ORDER BY name ASC
LIMIT 20;
```

### 3-4. GET /api/ingredients/stats/genre — ジャンル別統計

#### 処理フロー

1. JWT を検証する
2. `item_type = 1` の食材をジャンルごとに集計する
3. `genre IS NULL` の件数は `other` として集約する

#### DB操作

```sql
SELECT
  COALESCE(genre, 'other') AS genre,
  COUNT(*) AS count,
  ROUND(AVG(unit_price), 4) AS avg_unit_price
FROM items
WHERE item_type = 1
GROUP BY COALESCE(genre, 'other')
ORDER BY count DESC;
```

### 3-5. GET /api/ingredients/popular — 人気食材取得

#### クエリパラメータ

| パラメータ | デフォルト | 最大 |
|---|---|---|
| `limit` | 10 | 50 |

#### 処理フロー

1. JWT を検証する
2. `item_relations` テーブルで `child_id`（食材ID）の出現回数を集計する
3. 使用頻度の高い順に返す

#### DB操作

```sql
SELECT i.id, i.name, i.store, i.price, i.quantity, i.unit, i.unit_price, i.genre,
       COUNT(ir.child_id) AS usage_count
FROM items i
JOIN item_relations ir ON i.id = ir.child_id
WHERE i.item_type = 1
GROUP BY i.id
ORDER BY usage_count DESC
LIMIT :limit;
```

### 3-6. GET /api/ingredients/{id} — 食材詳細取得

#### 処理フロー

1. JWT を検証する
2. パスパラメータ `id` が正の整数であることを確認する
3. `SELECT * FROM items WHERE id = :id AND item_type = 1` を実行する
4. 取得できない場合は 404 を返す

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |
| `item_type != 1` のレコード（他種別との混同防止） | 404 | `NOT_FOUND` |

### 3-7. PUT /api/ingredients/{id} — 食材更新

#### バリデーションルール

| フィールド | 必須 | ルール |
|---|:---:|---|
| `name` | — | 1〜255文字 |
| `store_id` | — | 整数、ログインユーザーが所有する `stores.id`。送信時は `items.store` テキストも同期される。 |
| `price` | — | 数値、`>= 0` |
| `quantity` | — | 数値、`> 0` |
| `unit` | — | 1〜20文字 |
| `genre` | — | 50文字以内 |
| `description` | — | 任意文字列 |

#### 処理フロー

1. JWT を検証する
2. `SELECT id FROM items WHERE id = :id AND item_type = 1` で存在確認する（404 チェック）
3. リクエストボディをバリデーションする
4. `price` または `quantity` が送信された場合、`unit_price` を再計算する
   - 片方のみ送信された場合は、DBから現在値を取得してもう片方と組み合わせて再計算する
5. 変更フィールドのみ `UPDATE` する
6. 更新後のレコードを返す

#### DB操作

```sql
-- 存在確認
SELECT id, price, quantity FROM items WHERE id = :id AND item_type = 1;

-- 更新（変更フィールドのみ動的に構築）
UPDATE items
SET name = :name, store = :store, price = :price, quantity = :quantity,
    unit = :unit, unit_price = :unit_price, genre = :genre, description = :description,
    updated_at = NOW()
WHERE id = :id AND item_type = 1;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |
| バリデーション失敗 | 400 | `VALIDATION_ERROR` |

---

### 3-8. DELETE /api/ingredients/{id} — 食材削除

#### 処理フロー

1. JWT を検証する
2. `SELECT id FROM items WHERE id = :id AND item_type = 1` で存在確認する（404 チェック）
3. `item_relations` テーブルで `child_id = :id` のレコードが存在するか確認する（使用中チェック）
4. 使用中の場合は 409 を返す（メッセージに使用している仕込み品名を含める）
5. 使用中でない場合は `DELETE FROM items WHERE id = :id` を実行する

#### DB操作

```sql
-- 使用中チェック
SELECT ir.parent_id, i.name AS parent_name
FROM item_relations ir
JOIN items i ON ir.parent_id = i.id
WHERE ir.child_id = :id
LIMIT 1;

-- 削除
DELETE FROM items WHERE id = :id AND item_type = 1;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |
| 仕込み品・お品で使用中 | 409 | `CONFLICT` |

## 4. セキュリティ・パフォーマンス注意事項

- `sortBy` パラメータは許可リスト（`name`, `price`, `unit_price`, `store`, `created_at`, `updated_at`）で検証してからクエリに使用する（SQL インジェクション対策）
- LIKE 検索のワイルドカード文字（`%`, `_`, `\`）はエスケープしてからバインドパラメータに渡す
- `item_type = 1` の絞り込みを全クエリに必ず付ける（他種別レコードの混入防止）
- `idx_item_type`、`idx_name` インデックスを活用する
