## 3. 各エンドポイント処理詳細

### 3-1. GET /api/stores — 購入先一覧取得

#### 処理フロー

1. JWT を検証し `g.user_id` を取得する
2. `stores` テーブルから `user_id = g.user_id` のレコードを全件取得する
3. 各購入先に対して `items` テーブルで `item_type = 1 AND store_id = stores.id` の件数を取得し `ingredient_count` として付与する
4. `ingredient_count` 降順でソートして返す

#### DB操作

```sql
-- 購入先一覧
SELECT id, user_id, name, created_at, updated_at
FROM stores
WHERE user_id = :user_id;

-- 各購入先の使用食材数
SELECT COUNT(*) FROM items
WHERE item_type = 1 AND store_id = :store_id AND user_id = :user_id;
```

#### レスポンス

```json
{
  "success": true,
  "data": [
    { "id": 1, "name": "コスモス", "ingredient_count": 8 },
    { "id": 2, "name": "業務スーパー", "ingredient_count": 3 }
  ]
}
```

### 3-2. POST /api/stores — 購入先登録

#### バリデーションルール

| フィールド | 必須 | ルール |
|---|:---:|---|
| `name` | ✓ | 1〜100文字、空白のみ不可 |

#### 処理フロー

1. JWT を検証する
2. `name` をトリム・バリデーションする
3. `stores` テーブルで `user_id = g.user_id AND name = :name` の既存レコードを確認する
   - 存在する場合は 409 CONFLICT を返す
4. `stores` に INSERT する
5. `ingredient_count = 0` を付与して返す

#### DB操作

```sql
-- 重複チェック
SELECT id FROM stores WHERE user_id = :user_id AND name = :name LIMIT 1;

-- 登録
INSERT INTO stores (user_id, name, created_at, updated_at)
VALUES (:user_id, :name, NOW(), NOW());
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| `name` が空 | 400 | `VALIDATION_ERROR` |
| 同名購入先が存在 | 409 | `CONFLICT` |
| 認証失敗 | 401 | `UNAUTHORIZED` |

### 3-3. PUT /api/stores/{id} — 購入先更新

#### バリデーションルール

| フィールド | 必須 | ルール |
|---|:---:|---|
| `name` | ✓ | 1〜100文字、空白のみ不可 |

#### 処理フロー

1. JWT を検証する
2. `stores` で `id = :id AND user_id = g.user_id` を確認する（404 チェック）
3. `name` をトリム・バリデーションする
4. 同名の別購入先が存在するか確認する（409 チェック）
5. 旧購入先名（`old_name`）を保存する
6. `stores.name` を更新する
7. `items.store` を `old_name → new_name` で一括 UPDATE する（表示用テキストの同期）
8. 更新後の `ingredient_count` を取得して返す

#### DB操作

```sql
-- 存在確認
SELECT id, name FROM stores WHERE id = :id AND user_id = :user_id;

-- 重複チェック（自身を除く）
SELECT id FROM stores WHERE user_id = :user_id AND name = :new_name AND id != :id LIMIT 1;

-- 購入先名更新
UPDATE stores SET name = :new_name, updated_at = NOW() WHERE id = :id;

-- items.store の同期（後方互換）
UPDATE items SET store = :new_name
WHERE user_id = :user_id AND store = :old_name;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |
| `name` が空 | 400 | `VALIDATION_ERROR` |
| 同名別購入先が存在 | 409 | `CONFLICT` |

---

### 3-4. DELETE /api/stores/{id} — 購入先削除

#### 処理フロー

1. JWT を検証する
2. `stores` で `id = :id AND user_id = g.user_id` を確認する（404 チェック）
3. `items` で `item_type = 1 AND store_id = :id AND user_id = g.user_id` の件数を取得する
   - 1件以上存在する場合は 409 CONFLICT を返す（メッセージに件数を含める）
4. `stores` からレコードを DELETE する

#### DB操作

```sql
-- 存在確認
SELECT id, name FROM stores WHERE id = :id AND user_id = :user_id;

-- 使用中チェック
SELECT COUNT(*) FROM items
WHERE item_type = 1 AND store_id = :id AND user_id = :user_id;

-- 削除
DELETE FROM stores WHERE id = :id;
```

#### エラーパターン

| 条件 | ステータス | error コード |
|---|---|---|
| 存在しない id | 404 | `NOT_FOUND` |
| 食材で使用中 | 409 | `CONFLICT` |

---

## 4. セキュリティ注意事項

- 全エンドポイントで `user_id = g.user_id` の絞り込みを必ず行い、他ユーザーのデータへのアクセスを防止する
- PUT 時の `items.store` 一括更新も `user_id` で絞り込む
- `name` の入力値は `.strip()` でトリムしてから保存する
