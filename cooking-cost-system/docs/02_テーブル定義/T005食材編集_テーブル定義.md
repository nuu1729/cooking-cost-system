# 食材編集 テーブル定義

## 1. 概要
食材編集画面（`/ingredients/edit`）は、既存の食材マスタ（`ingredients` テーブル）のレコードを検索し、その内容（価格、量、単位、購入先など）を更新・更新履歴として管理するための機能を提供する。
本画面は主に既存テーブルに対する UPDATE 処理を行う。

## 2. 対象データ構造

| 対象テーブル | 処理内容 |
| :--- | :--- |
| **食材マスタ (`ingredients`)** | 選択された食材ID(`id`)に基づいてレコードを取得(GET)し、編集内容で上書き更新(PUT)を行う。 |
| **活動ログ (`activity_logs`)** | （※実装拡張時）食材の「変更前」「変更後」の比較情報をシステム内部で記録する場合に使用する履歴用テーブル。 |

## 3. テーブル詳細（`ingredients`）※更新対象カラム
食材追加時に作成されるカラムのうち、本画面から変更可能なカラムは以下の通りである。

| カラム名 (物理名) | データ型 | 更新の要否 | 備考 |
| :--- | :--- | :--- | :--- |
| `price` | DECIMAL(10,2) | 可能 | 取消線比較でUI表示される（変更前 / 変更後） |
| `quantity` | DECIMAL(10,2) | 可能 |
| `unit` | VARCHAR(20) | 可能 |
| `store` | VARCHAR(100) | 可能 |
| `unit_price` | DECIMAL(10,4) | 自動更新対象 | `price` や `quantity` が変更された場合、更新API内で再計算され上書きされる |
| `updated_at` | TIMESTAMP | 自動更新 | RDBMSの機能(ON UPDATE CURRENT_TIMESTAMP)により上書き時刻が記録される |

## 4. 特記事項・留意点
* **単価の再計算**: 価格や量のいずれかが変更された場合、必ず `price ÷ quantity` の再計算を行い、`unit_price` を最新状態に保つ必要がある。
* **関連データへの影響**: `ingredients` テーブルの内容（特に価格や単位）を更新した場合、それを材料として使用している「仕込み」や「完成品（お品）」の原価計算に波及する。本システムでは関連テーブル上でのコピー保持か、都度計算かをユースケースに応じて制御する。（後続の仕込みテーブル等では計算結果が都度反映されるか、コストをスナップショットで持つかに依存する）
