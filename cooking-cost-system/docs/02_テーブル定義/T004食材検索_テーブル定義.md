## 2. テーブル一覧

食材検索画面は **新規テーブルを作成しない**。  
登録済みの食材（`item_type = 1`）を参照するのみであり、データの登録（INSERT）は行わない。

| テーブル名 | 論理名 | 操作種別 | 概要 |
|---|---|:---:|---|
| `items` | 統合商品マスタ | 参照のみ | `item_type = 1`（食材）に絞って検索・比較・単位換算の元データとして利用 |

---

## 3. テーブル詳細（参照）

### 3-1. `items` テーブル — 食材検索で使用するカラム

> テーブルの全体定義は `T003 食材追加_テーブル定義.md` を参照。  
> 以下は食材検索処理で特に関係するカラムを抜粋して記載する。

| カラム名 | 論理名 | データ型 | 検索処理での役割 |
|---|---|---|---|
| `id` | 商品ID | `INT` | 結果行のトラッキングキー |
| `name` | 商品名 | `VARCHAR(255)` | 検索キーワードの照合対象（部分一致 `LIKE '%?%'`） |
| `store` | 購入先 | `VARCHAR(100)` | 比較リストでの購入先表示 |
| `price` | 価格 | `DECIMAL(10,2)` | 参考価格の表示 |
| `quantity` | 内容量 | `DECIMAL(10,2)` | 単位を揃えた比較計算の基準（例：100g 換算） |
| `unit` | 単位 | `VARCHAR(20)` | 量の単位を揃えた比較に使用 |
| `unit_price` | 単価 | `DECIMAL(10,4)` | 最安値のソート・特定（No.1 強調表示）に利用 |

#### 検索クエリの概要

```sql
-- 商品名による部分一致検索（食材のみ）
SELECT id, name, store, price, quantity, unit, unit_price
FROM   items
WHERE  item_type = 1
  AND  name LIKE :keyword
ORDER BY unit_price ASC;
```

> `:keyword` は `'%<入力値>%'` のパラメータとして渡す（SQL インジェクション防止）。

## 5. 補足事項

### 5-1. 集計・比較処理の実装方針

| 処理 | 実施場所 | 理由 |
|---|---|---|
| 100g 換算・単位変換 | API 層またはフロントエンド | 計算結果を永続化する必要がないため |
| 最安値のソート | DB（`ORDER BY unit_price ASC`） | インデックス活用で高速化 |
| 最安値の強調表示（No.1） | フロントエンド | UI ロジックのため |

### 5-2. 検索パフォーマンス

快適なリアルタイム検索（Debounce 制御付き）を提供するため、以下のインデックスを利用する。

| インデックス名 | 対象カラム | 効果 |
|---|---|---|
| `idx_name` | `name` | 商品名の部分一致検索を高速化 |
| `idx_item_type` | `item_type` | `item_type = 1` の絞り込みを高速化 |
| `idx_type_name` | `item_type`, `name` | 種別 + 名前の複合条件を高速化 |

> インデックス定義は `T003 食材追加_テーブル定義.md` §3-1 を参照。

### 5-3. セキュリティ

| 観点 | 方針 |
|---|---|
| SQL インジェクション | `name LIKE :keyword` のバインド変数で防止。ワイルドカード文字（`%`, `_`）はエスケープ処理を行うこと |
| 過剰なデータ取得 | ページネーション（`LIMIT / OFFSET`）を実装し、全件返却を防ぐ |
