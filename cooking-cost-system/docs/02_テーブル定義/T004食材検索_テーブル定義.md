# 食材検索 テーブル定義

## 1. 概要
食材検索画面（`/ingredients/search`）では、登録済みの食材を検索・比較・単位換算するための機能を提供する。
本画面はデータを参照するのみであり、新たなデータの登録（INSERT）機能は持たないため、本画面専用の新規テーブル構成は発生しない。

## 2. 参照するデータ構造
検索や比較表示を行うにあたり、以下のテーブルを参照する。

| テーブル名（物理名） | 用途 |
| :--- | :--- |
| **食材マスタ (`ingredients`)** | 食材の名称による検索、単価（`unit_price`）を用いた最安値計算、単位換算のための基準データとして読み込む。 |

## 3. テーブル詳細 (参照元)
本画面で主に利用される `ingredients` テーブルのカラムは以下の通りである。

| カラム名 | 説明 | 備考 |
| :--- | :--- | :--- |
| `name` | 検索キーワードの照合対象 | 前方一致または部分一致による曖昧検索に利用 |
| `store` | 候補リスト上の購入先表示 | 比較時の重要情報 |
| `price` | 参考価格の表示 |
| `quantity` / `unit` | 単位を揃えた比較計算の基準（例：100g換算） |
| `unit_price` | 単位あたりの単価 | 「最安値」のソート・特定（No.1強調表示）に利用 |

## 4. 特記事項
* **集計・比較処理**: 「量を揃える（100g換算など）」「単位を換算する（1個 → N g）」といった処理は、フロントエンド側もしくはAPI側の動的な計算ロジックによって実現されるため、計算結果を永続化するためのテーブル拡張等は行わない。
* **検索パフォーマンス**: 快適なリアルタイム検索（Debounce制御付き）を提供するため、本システムでは `ingredients` テーブルの `name` および `store` にインデックス（`idx_ingredients_search` 等）が付与されている。
