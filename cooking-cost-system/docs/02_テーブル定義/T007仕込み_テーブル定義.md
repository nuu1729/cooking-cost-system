# 仕込み テーブル定義

## 1. 概要
仕込み画面（`/preps` または `/dishes` ）は、登録済みの「食材（`ingredients`）」を組み合わせて、新たな半製品や料理の下地となる「仕込み（`dishes`）」を定義・保存するための画面である。
ここで保存された「仕込み」レコードと、そこに紐づく「構成食材」を保持するために、2つのテーブルを使用する。

## 2. テーブル一覧
| 論理名 | 物理名 | 役割・概要 |
| :--- | :--- | :--- |
| **料理マスタ (仕込み)** | `dishes` | 「仕込み名」「その仕込みで生じる総コスト」といった仕込み品（親）の基本情報を保持する。 |
| **料理-食材関連** | `dish_ingredients` | その仕込みを作るために、「どの食材(`ingredient_id`)」を「どれだけの量(`used_quantity`)」消費したかを結びつける（子）。 |

## 3. テーブル詳細

### 3.1 料理マスタ (`dishes`)
| カラム名 (物理名) | データ型 | NULL | 備考 |
| :--- | :--- | :--- | :--- |
| `id` | `INT` | ✕ | AUTO_INCREMENT (主キー) |
| `name` | `VARCHAR(255)` | ✕ | 仕込み名称（例：ごはん中、合わせ味噌） |
| `total_cost` | `DECIMAL(10,2)` | ✕ | 小計を全て足し合わせた合算金額（総コスト） |
| `genre` | `VARCHAR(50)` | ✕ | 仕込みカテゴリ（デフォルト: `meat`など） |
| `description` | `TEXT` | ◯ | この仕込みに関するメモや説明 |
| `created_at` | `TIMESTAMP` | ✕ | レコード作成時刻 |
| `updated_at` | `TIMESTAMP` | ✕ | レコード更新時刻 |

* **インデックス**: `idx_name` (`name`), `idx_total_cost` (`total_cost`), 主キー (`id`)

### 3.2 料理-食材関連 (`dish_ingredients`)
| カラム名 (物理名) | データ型 | NULL | 備考 |
| :--- | :--- | :--- | :--- |
| `id` | `INT` | ✕ | AUTO_INCREMENT (主キー) |
| `dish_id` | `INT` | ✕ | 外部キー：紐づく仕込みの親ID (`dishes.id`) |
| `ingredient_id`| `INT` | ✕ | 外部キー：紐づく使用食材のID (`ingredients.id`) |
| `used_quantity`| `DECIMAL(10,2)`| ✕ | この仕込みにおける食材の使用量 |
| `used_cost` | `DECIMAL(10,2)`| ✕ | 選択食材の単価 × `used_quantity` の算出結果 |
| `created_at` | `TIMESTAMP` | ✕ | レコード作成時刻 |

* **制約・インデックス**:
  - `fk_dish_ingredients_dish` (`dish_id` -> `dishes(id)`)
  - `fk_dish_ingredients_ingredient` (`ingredient_id` -> `ingredients(id)`)
  - 重複登録防止の複合ユニークキー `uk_dish_ingredient` (`dish_id`, `ingredient_id`)

## 4. DDL（CREATE TABLE文）
```sql
-- 料理(仕込み)テーブル
CREATE TABLE `dishes` (
    `id` INT NOT NULL AUTO_INCREMENT,
    `name` VARCHAR(255) NOT NULL COMMENT '料理(仕込み)名',
    `total_cost` DECIMAL(10,2) NOT NULL COMMENT '総コスト',
    `genre` VARCHAR(50) NOT NULL DEFAULT 'meat' COMMENT 'ジャンル',
    `description` TEXT NULL COMMENT '説明',
    `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '作成日時',
    `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新日時',
    PRIMARY KEY (`id`),
    INDEX `idx_name` (`name`),
    INDEX `idx_genre` (`genre`),
    INDEX `idx_created_at` (`created_at`),
    INDEX `idx_total_cost` (`total_cost`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='料理(仕込み)マスタ';

-- 料理-食材関連テーブル
CREATE TABLE `dish_ingredients` (
    `id` INT NOT NULL AUTO_INCREMENT,
    `dish_id` INT NOT NULL COMMENT '料理ID',
    `ingredient_id` INT NOT NULL COMMENT '食材ID',
    `used_quantity` DECIMAL(10,2) NOT NULL COMMENT '使用量',
    `used_cost` DECIMAL(10,2) NOT NULL COMMENT '使用コスト',
    `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '作成日時',
    PRIMARY KEY (`id`),
    CONSTRAINT `fk_dish_ingredients_dish` 
        FOREIGN KEY (`dish_id`) REFERENCES `dishes` (`id`) 
        ON DELETE RESTRICT ON UPDATE CASCADE,
    CONSTRAINT `fk_dish_ingredients_ingredient` 
        FOREIGN KEY (`ingredient_id`) REFERENCES `ingredients` (`id`) 
        ON DELETE RESTRICT ON UPDATE CASCADE,
    INDEX `idx_dish` (`dish_id`),
    INDEX `idx_ingredient` (`ingredient_id`),
    UNIQUE KEY `uk_dish_ingredient` (`dish_id`, `ingredient_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='料理-食材関連';
```

## 5. 補足事項・特記事項
* **合計コスト（`total_cost`）の保持**:
  画面上（右側カラム）で計算される全食材値段の合計値は、仕込み保存時に `dishes.total_cost` へと書き込まれる。これにより、一覧表示や他への流用時に毎回の合算計算（SUMなど）を省くことができる。
* **トランザクション管理**:
  仕込みの登録（画面上の「仕込み確定」）時は、`dishes` テーブルへの INSERT（親の作成） と、複数の `dish_ingredients` への INSERT（構成食材群の登録） を同時に行う必要がある。そのため、バックエンドAPIはこれらの操作を一連の **トランザクション処理** として実行し、途中でエラーが起きた場合はロールバックしてデータの整合性を保つ。（※本ドキュメント内のDB構造では `ON DELETE CASCADE` ではなく、アプリケーション側でバリデーションを行える `RESTRICT` 等を意図した設計要件となる場合がある）
