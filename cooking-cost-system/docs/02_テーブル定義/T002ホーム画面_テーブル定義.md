# ホーム画面 テーブル定義

## 1. 概要
ホーム画面（`/`）独自のデータ永続化は発生しないため、本画面専用の新規テーブル定義は不要である。
画面の背景画像やユーザー情報などは、アカウント情報（及びログイン情報）に依存する。

## 2. 参照するデータ構造
ホーム画面のUI構築に必要なデータは、基本的にクライアントサイドのローカルストレージ（`accountStore`）、または認証時のコンテキスト（JWT等）から取得される。

| データ項目 | 取得元 | 用途 |
| :--- | :--- | :--- |
| **ホーム背景画像 (`homeBgDataUrl`)** | クライアント `accountStore` (LocalStorage) | 画面右側の背景画像として表示。未設定時はデフォルト画像(`ming_outlook.jpeg`)を表示する。 |
| **ユーザー表示名 (`displayName`)** | クライアント `accountStore` (LocalStorage) | 将来的にウェルカムメッセージ等で表示する用途（現在設計では使用対象外） |

## 3. DDL (不要)
※本画面のための専用テーブルは存在しないため、DDLの記述は省略する。

## 4. 補足事項
* 背景画像は現在Base64エンコード形式でLocalStorageに保存されているが、将来的に画像の実体をクラウドストレージ（S3等）にアップロードし、そのURLを `users` テーブル等に保持する構成に変更する可能性がある。その場合、ユーザーのアカウント情報テーブル（例:`users.home_bg_url`）を追加する。
