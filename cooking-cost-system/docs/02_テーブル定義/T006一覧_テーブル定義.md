# 一覧 テーブル定義

## 1. 概要
一覧画面（`/ingredients/list` 等）は、システム上に登録されている各種マスタデータ（食材、仕込み、お品）をタブ切り替えで一元的に表示・検索・削除するための画面である。
画面内で新たなテーブルは作成されないが、システムが持つ3種類の主要なマスターテーブルを横断的に参照し、削除時（DELETE）の整合性を扱うための重要な接点となる。

## 2. 参照・操作するテーブル一覧

| カテゴリ | 対象テーブル (物理名) | 操作 | 用途 |
| :--- | :--- | :--- | :--- |
| **食材** | `ingredients` | GET / DELETE | 登録済みの食材の一覧表示、検索、削除機能 |
| **仕込み** | `dishes` | GET / DELETE | 登録済みの仕込み（半製品）の一覧表示、検索、削除 |
| **お品**   | `completed_foods` | GET / DELETE | 登録済みの完成品（メニュー）の一覧表示、検索、削除 |

## 3. 主要なカラム詳細 (各種マスタ)
一覧画面のテーブル行(Table Body)に表示する共通および必須となるカラム。

### 3.1 食材タブ (`ingredients`)
- `id`: トラッキング、削除キー用
- `name`: 食材名（詳細へのリンク・検索対象）
- `unit_price`: 単価表示用（例：¥0.48/g）
- `store`: 購入先表示用

### 3.2 仕込みタブ (`dishes`)
- `id`: トラッキング、削除キー用
- `name`: 仕込み名（詳細へのリンク・検索対象）
- `total_cost`: 単価・計算コスト表示用
- `description` / `genre`: 概要表示（必要に応じて）

### 3.3 お品タブ (`completed_foods`)
- `id`: トラッキング、削除キー用
- `name`: お品名（詳細へのリンク・検索対象）
- `price` / `total_cost`: 合計金額・原価表示用

## 4. 削除機能における制約（外部キー・バリデーション制御）
一覧画面の削除アイコン押下時、RDBMS上の **外部キー制約（Foreign Key）** に基づいたエラーハンドリングが必須となる。マスターデータを不用意に削除することで、それらに依存している「仕込み」や「お品」の構成が破損するのを防ぐためである。

1. **食材（`ingredients`）の削除要件**:
   - `dish_ingredients`（仕込みの構成要素）として1件でも紐づいている食材は、原則として削除不可とする。（エラーメッセージ：「この食材は『仕込みA』で使用中のため削除できません」）
   - RDBMSの設定で `ON DELETE CASCADE` が付与されている場合、親となる `ingredients` を消すと依存する紐付きが連鎖削除されるため、**アプリケーション層またはAPIレベルでの事前チェックによる削除拒否（制限）** を推奨する。

2. **仕込み（`dishes`）の削除要件**:
   - `food_dishes`（お品の構成要素）として紐づいている仕込みは、削除不可とする。

3. **リスト取得（一覧表示）時の要件**:
   - 辞書順（昇順）での表示がデフォルト動作となるため、API側の `ORDER BY name ASC` 取得クエリが実行される。
