## 3. テーブル定義

### 3-1. `users` テーブル（ユーザーマスタ）

#### カラム定義

| カラム名 | 論理名 | データ型 | NOT NULL | デフォルト値 | 備考 |
|---|---|---|:---:|---|---|
| `id` | ユーザーID | `INT` | ✓ | AUTO_INCREMENT | 主キー |
| `email` | メールアドレス | `VARCHAR(255)` | ✓ | — | ユニーク制約あり。ログイン識別子 |
| `password_hash` | パスワードハッシュ | `VARCHAR(255)` | ✓ | — | bcrypt等でハッシュ化したパスワードを格納 |
| `username` | アカウント名 | `VARCHAR(50)` | — | `NULL` | 任意入力。最大50文字 |
| `role` | ロール | `ENUM('user', 'admin')` | ✓ | `'user'` | 権限区分。現フェーズは `user` のみ使用 |
| `is_active` | 有効フラグ | `TINYINT(1)` | ✓ | `1` | `1`: 有効 / `0`: 無効（論理削除用） |
| `last_login` | 最終ログイン日時 | `DATETIME` | — | `NULL` | ログイン成功時に更新する |
| `created_at` | 作成日時 | `TIMESTAMP` | ✓ | `CURRENT_TIMESTAMP` | レコード作成日時（自動設定） |
| `updated_at` | 更新日時 | `TIMESTAMP` | ✓ | `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | レコード更新日時（自動更新） |

#### 制約・インデックス定義

| 制約名 | 種別 | 対象カラム | 説明 |
|---|---|---|---|
| `PRIMARY KEY` | 主キー | `id` | 主キー制約 |
| `uk_email` | ユニークキー | `email` | メールアドレスの重複登録を防止 |
| `idx_email` | インデックス | `email` | メールアドレスによる検索を高速化 |
| `idx_is_active` | インデックス | `is_active` | 有効ユーザーの絞り込みを高速化 |
| `idx_created_at` | インデックス | `created_at` | 作成日時による並べ替え・絞り込みを高速化 |

## 4. DDL（CREATE TABLE文）

```sql
-- ユーザーマスタテーブル
CREATE TABLE `users` (
    `id`            INT             NOT NULL AUTO_INCREMENT                              COMMENT 'ユーザーID',
    `email`         VARCHAR(255)    NOT NULL                                             COMMENT 'メールアドレス（ログイン識別子）',
    `password_hash` VARCHAR(255)    NOT NULL                                             COMMENT 'パスワードハッシュ（bcrypt）',
    `username`      VARCHAR(50)     NULL      DEFAULT NULL                               COMMENT 'アカウント名（任意）',
    `role`          ENUM('user', 'admin')
                                    NOT NULL  DEFAULT 'user'                             COMMENT 'ロール（user / admin）',
    `is_active`     TINYINT(1)      NOT NULL  DEFAULT 1                                  COMMENT '有効フラグ（1: 有効, 0: 無効）',
    `last_login`    DATETIME        NULL      DEFAULT NULL                               COMMENT '最終ログイン日時',
    `created_at`    TIMESTAMP       NOT NULL  DEFAULT CURRENT_TIMESTAMP                  COMMENT '作成日時',
    `updated_at`    TIMESTAMP       NOT NULL  DEFAULT CURRENT_TIMESTAMP
                                              ON UPDATE CURRENT_TIMESTAMP                COMMENT '更新日時',

    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_email`       (`email`),
    INDEX       `idx_email`     (`email`),
    INDEX       `idx_is_active` (`is_active`),
    INDEX       `idx_created_at`(`created_at`)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8mb4
  COLLATE=utf8mb4_unicode_ci
  COMMENT='ユーザーマスタ';
```

## 5. 補足事項

### 5-1. パスワード管理方針

| 項目 | 内容 |
|---|---|
| ハッシュアルゴリズム | bcrypt（コストファクター: 12推奨） |
| 保存列 | `password_hash`（平文は絶対に保存しない） |
| 複雑性ルール | 8文字以上・大文字・小文字・数字をすべて含むこと |

### 5-2. JWT認証との関係

- アカウント作成成功（`201 Created`）時にJWTトークンを発行し、レスポンスに含める。  
- 発行したトークンにはペイロードとして `id`・`email`・`role` を含める（実装は `services` 層で行う）。  
- トークン自体はDBには保存しない（ステートレス運用）。

### 5-3. アカウント編集画面との関係

| 画面操作 | 対象カラム | API |
|---|---|---|
| アカウント名変更 | `username` | `PUT /api/auth/profile` |
| メールアドレス変更 | `email` | `PUT /api/auth/profile` |
| パスワード変更 | `password_hash` | `PUT /api/auth/password` |
| 最終ログイン記録 | `last_login` | ログイン処理内で自動更新 |

### 5-4. 将来の拡張予定

| 項目 | 対応案 |
|---|---|
| アカウントアイコン | `avatar_url VARCHAR(512)` カラムを追加予定（現フェーズは localStorage に base64 で保存） |
| 論理削除 | `is_active = 0` で対応済み。退会機能実装時に利用する |
| OAuth連携 | 外部テーブル（`oauth_providers` 等）を追加し、`users.id` に外部キーで紐付ける予定 |
| 2段階認証 | `totp_secret` カラム等を追加して対応予定 |
