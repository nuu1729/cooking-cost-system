# 食材追加 テーブル定義

## 1. 概要
食材追加画面（`/ingredients/add`）から入力された「食材・調味料」の購入情報を保存するための `ingredients` テーブルを定義する。
本テーブルは、料理の原価計算の基盤となるもっとも細かいマスターデータとして機能する。

## 2. テーブル一覧
| 論理名 | 物理名 | 役割・概要 |
| :--- | :--- | :--- |
| **食材マスタ** | `ingredients` | 仕入れた食材の名前、購入先、価格、量、および単位を保持する。 |

## 3. テーブル詳細（`ingredients`）

### 3.1. カラム定義
| 論理名 | 物理名 | データ型 | NULL | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **食材ID** | `id` | `INT` | ✕ | AUTO_INCREMENT | 主キー |
| **食材名** | `name` | `VARCHAR(255)` | ✕ | なし | 食材の名称（画面入力項目：商品名） |
| **購入先** | `store` | `VARCHAR(100)` | ✕ | なし | 仕入れ先やスーパー名（画面入力項目：購入先） |
| **購入量** | `quantity` | `DECIMAL(10,2)` | ✕ | なし | 購入した量（画面入力項目：量） |
| **単位** | `unit` | `VARCHAR(20)`| ✕ | なし | g, ml, 個 など（画面入力項目：単位） |
| **購入価格**| `price` | `DECIMAL(10,2)` | ✕ | なし | 支払い金額（画面入力項目：価格） |
| **単価**   | `unit_price` | `DECIMAL(10,4)` | ✕ | なし | `price / quantity` の自動計算値。原価計算で使用 |
| **ジャンル**| `genre` | `ENUM('meat','vegetable','seasoning','sauce','frozen','drink')` | ✕ | `'meat'` | 食材の分類。※画面設計仕様段階では入力フォームなし、現在は内部でデフォルト値を設定 |
| **作成日時** | `created_at` | `TIMESTAMP` | ✕ | CURRENT_TIMESTAMP | レコード作成時刻 |
| **更新日時** | `updated_at` | `TIMESTAMP` | ✕ | CURRENT_TIMESTAMP... | レコード更新時刻 |

### 3.2. 制約・インデックス
| 種類 | 対象カラム | 名称 | 説明 |
| :--- | :--- | :--- | :--- |
| 🎈 主キー | `id` | `PRIMARY` | 一意性を保証する連番 |
| 🔑 ユニークキー | `name`, `store`, `unit`| `uk_ingredient` | 同じ名前・購入先・単位の食材の重複登録を防ぐ |
| 🔍 インデックス | `name` | `idx_name` | 食材名による検索の高速化 |
| 🔍 インデックス | `store` | `idx_store` | 購入先による検索の高速化 |
| 🔍 インデックス | `genre` | `idx_genre` | ジャンル絞り込み用 |
| 🔍 インデックス | `created_at` | `idx_created_at` | 新着順ソートの高速化 |
| 🔍 インデックス | `unit_price`| `idx_unit_price` | 単価比較分析用のインデックス |
| 複合インデックス | `name`, `store` | `idx_ingredients_search` | 食材名と購入先による複合検索のパフォーマンス向上用 |

## 4. DDL（CREATE TABLE文）
```sql
CREATE TABLE `ingredients` (
    `id` INT NOT NULL AUTO_INCREMENT,
    `name` VARCHAR(255) NOT NULL COMMENT '食材名',
    `store` VARCHAR(100) NOT NULL COMMENT '購入場所',
    `quantity` DECIMAL(10,2) NOT NULL COMMENT '購入量',
    `unit` VARCHAR(20) NOT NULL COMMENT '単位',
    `price` DECIMAL(10,2) NOT NULL COMMENT '購入価格',
    `unit_price` DECIMAL(10,4) NOT NULL COMMENT '単価（自動計算）',
    `genre` ENUM('meat', 'vegetable', 'seasoning', 'sauce', 'frozen', 'drink') NOT NULL DEFAULT 'meat' COMMENT 'ジャンル',
    `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '作成日時',
    `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新日時',
    PRIMARY KEY (`id`),
    INDEX `idx_name` (`name`),
    INDEX `idx_store` (`store`),
    INDEX `idx_genre` (`genre`),
    INDEX `idx_created_at` (`created_at`),
    INDEX `idx_unit_price` (`unit_price`),
    UNIQUE KEY `uk_ingredient` (`name`, `store`, `unit`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='食材マスタ';

-- 複合検索用インデックス
CREATE INDEX `idx_ingredients_search` ON `ingredients`(`name`, `store`);
```

## 5. 補足事項・特記事項
* **単価 (`unit_price`)**: 料理作成時の原価計算を高速に行うため、登録時・更新時に `価格(price) ÷ 購入量(quantity)` で算出された値（小数点第4位まで）を保存する。
* **重複登録 (UPSERT の考慮)**: 同じ食材名・購入先・単位での重複登録が行われた場合、既存の入力行を更新（価格や量を上書き）するか、エラーとして弾くかの制御はアプリケーション側の設計で行う。
* **音声入力の対応**: 画面からの音声入力をパースした結果も、最終的には上記のフィールドにマッピングされ、通常のテキスト入力と同じように扱われて登録される。
