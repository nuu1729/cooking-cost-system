# 統合商品マスタ (items) テーブル定義
（旧：食材追加テーブル定義）

## 1. 概要
食材、仕込み品、お品（完成品）など、システム内で扱うすべての「商品・物品」を統一的に管理するための `items` テーブルを定義する。
従来の `ingredients`, `dishes`, `completed_foods` 等のテーブルを統合し、BOM（部品表）構造の土台となるもっとも重要なマスターデータである。
「食材追加」「仕込み」「お品」の各画面から作成されるデータはすべてこのテーブルに保存され `item_type` によって種類が区別される。

## 2. テーブル一覧
| 論理名 | 物理名 | 役割・概要 |
| :--- | :--- | :--- |
| **統合商品マスタ** | `items` | すべての品目の基本情報（名前、属性、購入先、価格、量、単位など）を一元管理する。 |

## 3. テーブル詳細（`items`）

### 3.1. カラム定義
| 論理名 | 物理名 | データ型 | NULL | デフォルト値 | 説明 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **商品ID** | `id` | `INT` | ✕ | AUTO_INCREMENT | 主キー |
| **商品名** | `name` | `VARCHAR(255)` | ✕ | なし | 品目の名称（食材名、仕込み名、お品名） |
| **属性タイプ**| `item_type` | `TINYINT` | ✕ | なし | 1: 食材, 2: 仕込み品, 3: お品 |
| **購入/調達先**| `store` | `VARCHAR(100)` | ✕ | なし | 購入先、スーパー名等（仕込み・お品の場合は「自家製」等） |
| **原価/価格**| `price` | `DECIMAL(10,2)` | ✕ | なし | 支払い金額、または算出された総コスト |
| **内容量/完成量**| `quantity` | `DECIMAL(10,2)` | ✕ | なし | 購入した量、または1回で作れる仕上がり量 |
| **単位** | `unit` | `VARCHAR(20)`| ✕ | なし | g, ml, 個 など |
| **単価**   | `unit_price` | `DECIMAL(10,4)` | ✕ | なし | `price / quantity` の自動計算値。原価計算で使用 |
| **ジャンル**| `genre` | `VARCHAR(50)` | ◯ | NULL | 食材の分類や料理のジャンル用 |
| **説明/メモ**| `description`| `TEXT` | ◯ | NULL | 調理方法や特記事項のメモ |
| **作成日時** | `created_at` | `TIMESTAMP` | ✕ | CURRENT_TIMESTAMP | レコード作成時刻 |
| **更新日時** | `updated_at` | `TIMESTAMP` | ✕ | CURRENT_TIMESTAMP... | レコード更新時刻 |

### 3.2. 制約・インデックス
| 種類 | 対象カラム | 名称 | 説明 |
| :--- | :--- | :--- | :--- |
| 🎈 主キー | `id` | `PRIMARY` | 一意性を保証する連番 |
| 🔍 インデックス | `item_type` | `idx_item_type` | タブ切り替え・検索用 |
| 🔍 インデックス | `name` | `idx_name` | 商品名による検索用 |
| 🔍 インデックス | `store` | `idx_store` | 購入先による検索用 |
| 複合インデックス | `item_type`, `name` | `idx_type_name` | 種類別一覧でのソート・検索の高速化 |

## 4. DDL（CREATE TABLE文）
```sql
CREATE TABLE `items` (
    `id` INT NOT NULL AUTO_INCREMENT,
    `name` VARCHAR(255) NOT NULL COMMENT '商品名',
    `item_type` TINYINT NOT NULL COMMENT '属性タイプ(1:食材,2:仕込み,3:お品)',
    `store` VARCHAR(100) NOT NULL COMMENT '購入場所/調達先',
    `price` DECIMAL(10,2) NOT NULL COMMENT '価格/総コスト',
    `quantity` DECIMAL(10,2) NOT NULL COMMENT '内容量/仕上がり量',
    `unit` VARCHAR(20) NOT NULL COMMENT '単位',
    `unit_price` DECIMAL(10,4) NOT NULL COMMENT '単価（自動計算）',
    `genre` VARCHAR(50) NULL COMMENT 'ジャンル',
    `description` TEXT NULL COMMENT '説明・メモ',
    `created_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '作成日時',
    `updated_at` TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新日時',
    PRIMARY KEY (`id`),
    INDEX `idx_item_type` (`item_type`),
    INDEX `idx_name` (`name`),
    INDEX `idx_store` (`store`),
    INDEX `idx_type_name` (`item_type`, `name`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci COMMENT='統合商品マスタ';
```

## 5. 補足事項・特記事項
* **単価 (`unit_price`)**: 料理作成時の原価計算を高速に行うため、保存時に `価格(price) ÷ 量(quantity)` で算出された値（小数点第4位まで）を保存する。
* **統一管理の利点**: 従来の画面ごとにバラバラだったテーブルを1つに統合したことで、「一覧」画面では `SELECT * FROM items` を行いフロントエンドで `item_type` に応じてタブを分けるなどの極めてシンプルな実装が可能。
