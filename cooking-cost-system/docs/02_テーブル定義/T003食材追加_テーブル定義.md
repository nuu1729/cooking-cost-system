## 2. テーブル一覧

食材・仕込み品・お品（完成品）など、システム内で扱うすべての品目を統一的に管理する。  
従来の `ingredients` / `dishes` / `completed_foods` を統合した BOM（部品表）設計の中核テーブル。

| テーブル名 | 論理名 | 説明 |
|---|---|---|
| `items` | 統合商品マスタ | すべての品目の基本情報を `item_type` で種別管理する |

## 3. テーブル定義

### 3-1. `items` テーブル（統合商品マスタ）

#### カラム定義

| カラム名 | 論理名 | データ型 | NOT NULL | デフォルト値 | 備考 |
|---|---|---|:---:|---|---|
| `id` | 商品ID | `INT` | ✓ | AUTO_INCREMENT | 主キー |
| `name` | 商品名 | `VARCHAR(255)` | ✓ | — | 食材名・仕込み名・お品名 |
| `item_type` | 属性タイプ | `TINYINT` | ✓ | — | 1: 食材 / 2: 仕込み品 / 3: お品。CHECK 制約あり |
| `store` | 購入/調達先 | `VARCHAR(100)` | ✓ | — | 食材はスーパー名等、仕込み・お品は `'自家製'` 等を入力 |
| `price` | 原価/価格 | `DECIMAL(10,2)` | ✓ | — | 支払い金額または算出された総コスト。`>= 0` CHECK あり |
| `quantity` | 内容量/完成量 | `DECIMAL(10,2)` | ✓ | — | 購入した量または1回で作れる仕上がり量。`> 0` CHECK あり |
| `unit` | 単位 | `VARCHAR(20)` | ✓ | — | g / ml / 個 など |
| `unit_price` | 単価 | `DECIMAL(10,4)` | ✓ | — | `price ÷ quantity` の計算値。保存時にアプリ層で算出して格納 |
| `genre` | ジャンル | `VARCHAR(50)` | — | `NULL` | 食材の分類・料理ジャンル（任意） |
| `description` | 説明/メモ | `TEXT` | — | `NULL` | 調理メモや特記事項（任意） |
| `created_at` | 作成日時 | `TIMESTAMP` | ✓ | `CURRENT_TIMESTAMP` | レコード作成日時（自動設定） |
| `updated_at` | 更新日時 | `TIMESTAMP` | ✓ | `CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | レコード更新日時（自動更新） |

> **設計方針（user_id について）**  
> 現フェーズでは `user_id` カラムを持たない。本システムは **単一テナント（1ユーザー/1店舗）** を前提としており、データはシステム全体で共有する。  
> 将来マルチテナント対応が必要になった場合は `user_id INT NOT NULL` カラムを追加し、`users.id` への外部キーと `(user_id, item_type, name)` の複合インデックスを設ける。

#### 制約・インデックス定義

| 制約名 | 種別 | 対象カラム | 説明 |
|---|---|---|---|
| `PRIMARY KEY` | 主キー | `id` | 主キー制約 |
| `chk_item_type` | CHECK | `item_type` | 値は 1・2・3 のみ許可 |
| `chk_price` | CHECK | `price` | `price >= 0` |
| `chk_quantity` | CHECK | `quantity` | `quantity > 0` |
| `idx_item_type` | インデックス | `item_type` | タブ切り替え・種別絞り込みの高速化 |
| `idx_name` | インデックス | `name` | 商品名検索の高速化 |
| `idx_store` | インデックス | `store` | 購入先検索の高速化 |
| `idx_type_name` | 複合インデックス | `item_type`, `name` | 種別別一覧のソート・検索の高速化 |

## 4. DDL（CREATE TABLE文）

```sql
CREATE TABLE `items` (
    `id`          INT              NOT NULL AUTO_INCREMENT                              COMMENT '商品ID',
    `name`        VARCHAR(255)     NOT NULL                                             COMMENT '商品名（食材名・仕込み名・お品名）',
    `item_type`   TINYINT          NOT NULL                                             COMMENT '属性タイプ（1:食材 2:仕込み品 3:お品）',
    `store`       VARCHAR(100)     NOT NULL                                             COMMENT '購入先/調達先（仕込み・お品は「自家製」等）',
    `price`       DECIMAL(10,2)    NOT NULL                                             COMMENT '価格/総コスト（>= 0）',
    `quantity`    DECIMAL(10,2)    NOT NULL                                             COMMENT '内容量/仕上がり量（> 0）',
    `unit`        VARCHAR(20)      NOT NULL                                             COMMENT '単位（g / ml / 個 等）',
    `unit_price`  DECIMAL(10,4)    NOT NULL                                             COMMENT '単価（price ÷ quantity、アプリ層で算出）',
    `genre`       VARCHAR(50)      NULL      DEFAULT NULL                               COMMENT 'ジャンル（任意）',
    `description` TEXT             NULL      DEFAULT NULL                               COMMENT '説明・メモ（任意）',
    `created_at`  TIMESTAMP        NOT NULL  DEFAULT CURRENT_TIMESTAMP                  COMMENT '作成日時',
    `updated_at`  TIMESTAMP        NOT NULL  DEFAULT CURRENT_TIMESTAMP
                                             ON UPDATE CURRENT_TIMESTAMP                COMMENT '更新日時',

    PRIMARY KEY (`id`),
    CONSTRAINT `chk_item_type` CHECK (`item_type` IN (1, 2, 3)),
    CONSTRAINT `chk_price`     CHECK (`price` >= 0),
    CONSTRAINT `chk_quantity`  CHECK (`quantity` > 0),
    INDEX `idx_item_type` (`item_type`),
    INDEX `idx_name`      (`name`),
    INDEX `idx_store`     (`store`),
    INDEX `idx_type_name` (`item_type`, `name`)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8mb4
  COLLATE=utf8mb4_unicode_ci
  COMMENT='統合商品マスタ（食材/仕込み品/お品を item_type で区別）';
```

## 5. 補足事項

### 5-1. item_type の値定義

| 値 | 種別 | 作成画面 | store の入力値例 |
|---|---|---|---|
| `1` | 食材 | 食材追加（`/ingredients/add`） | `'コストコ'` / `'業務スーパー'` |
| `2` | 仕込み品 | 仕込み（`/preps`） | `'自家製'` |
| `3` | お品（完成品） | お品（`/foods`） | `'自家製'` |

### 5-2. unit_price の管理方針

`unit_price` はアプリケーション層（API の保存処理）で `price ÷ quantity` を計算して格納する。  
DB の GENERATED COLUMN を使う方法も有効だが、現フェーズはアプリ層での計算を採用する。  
`price` または `quantity` を更新した場合は必ず `unit_price` も再計算して同時に UPDATE すること（T005 参照）。

### 5-3. セキュリティ

| 観点 | 方針 |
|---|---|
| SQL インジェクション | すべての DB 操作はパラメータライズドクエリ（ORM のバインド変数）を使用すること。生クエリへの文字列結合は禁止 |
| 入力バリデーション | `price` / `quantity` は API 層で数値チェック・範囲チェックを実施し、CHECK 制約と二重防衛する |
| `item_type` の改ざん | API レイヤーで許可値（1/2/3）を検証する。フロントエンドからの値をそのまま信用しない |

### 5-4. 将来の拡張予定

| 項目 | 対応案 |
|---|---|
| マルチテナント化 | `user_id INT NOT NULL` カラム追加 + `users.id` への外部キー |
| 論理削除 | `deleted_at DATETIME NULL` カラム追加（現フェーズは物理削除） |
| 単価の自動計算 | `unit_price` を `GENERATED COLUMN` に変更（DB 側で一貫性を保証） |
| バーコード連携 | `barcode VARCHAR(30) NULL` カラム追加 |
