## 2. テーブル一覧

販売価格シミュレーターは **シミュレーション専用画面** であり、計算結果の永続化は行わない。  
画面仕様書 §7 に明記されている通り「データ保存は行わない」。

| テーブル名 | 論理名 | 操作種別 | 概要 |
|---|---|:---:|---|
| `items` | 統合商品マスタ | 参照のみ | `item_type = 3`（お品）の一覧および食材原価（`price`）を取得する |

---

## 3. テーブル詳細（参照）

### 3-1. `items` テーブル — シミュレーターで使用するカラム

> テーブルの全体定義は `T003 食材追加_テーブル定義.md` を参照。  
> 以下はシミュレーター処理で特に関係するカラムを抜粋して記載する。

| カラム名 | 論理名 | データ型 | シミュレーターでの役割 |
|---|---|---|---|
| `id` | 商品ID | `INT` | お品選択セレクトボックスの value |
| `name` | お品名 | `VARCHAR(255)` | お品選択セレクトボックスの表示名 |
| `item_type` | 属性タイプ | `TINYINT` | `= 3` でお品に絞り込む |
| `price` | 食材原価 | `DECIMAL(10,2)` | シミュレーターの「食材原価」として読み取り専用で表示（自動セット） |

#### お品一覧取得クエリ

```sql
-- お品選択セレクトボックスの選択肢を生成
SELECT id, name, price
FROM   items
WHERE  item_type = 3
ORDER BY name ASC;
```

## 4. 計算ロジック（サーバーレス・フロントエンド完結）

すべての計算はフロントエンドまたは API 層のメモリ上で行い、DB への書き込みは発生しない。

### 4-1. 主要指標の計算式

| 指標 | 計算式 | 単位 |
|---|---|---|
| 原価率（Food Cost Ratio） | `食材原価 ÷ 販売価格 × 100` | % |
| 粗利益（Gross Profit） | `販売価格 − 食材原価` | 円 |
| 粗利益率（Gross Profit Margin） | `粗利益 ÷ 販売価格 × 100` | % |
| FL 比率（推定） | `(食材費 + 推定人件費30%) ÷ 販売価格 × 100` | % |
| 推奨販売価格 | `食材原価 ÷ 目標原価率`（10円単位切り上げ） | 円 |
| コスト変動後原価率 | `食材原価 × (1 + 上昇率) ÷ 販売価格 × 100` | % |

### 4-2. 判定基準

| マーク | 判定 | 原価率の範囲 |
|---|---|---|
| ◎ 最適 | `< 30%` | 三分法の目標以内 |
| ○ 適正 | `30% 以上 35% 未満` | 業界標準範囲内 |
| △ 要注意 | `35% 以上 50% 未満` | 収益悪化の可能性あり |
| ✗ 赤字リスク | `>= 50%` | 価格改定を強く推奨 |

## 6. 補足事項

### 6-1. セキュリティ

| 観点 | 方針 |
|---|---|
| 権限チェック | API 層で JWT を検証し、認証済みユーザーのみ `items` を参照できる |
| お品以外の参照防止 | `WHERE item_type = 3` を必ず付与し、食材・仕込み品の price 情報が混入しないようにする |
| 計算結果の信頼性 | 販売価格・目標原価率はフロントエンドから渡された値をサーバー側で再計算して返す（フロント側計算結果をそのまま信用しない） |

### 6-2. 将来の拡張予定

| 項目 | 対応案 |
|---|---|
| 目標原価率の永続化 | `system_settings` テーブル（`user_id`, `key`, `value`, `updated_at`）を追加し、ユーザーごとに設定値を保存できるようにする |
| シミュレーション履歴 | `simulation_logs` テーブル（`user_id`, `item_id`, `selling_price`, `cost_ratio`, `result`, `created_at`）を追加して過去の診断結果を参照できるようにする |

#### 将来追加予定テーブル: `system_settings`

```sql
-- （参考 DDL：現フェーズでは作成しない）
CREATE TABLE `system_settings` (
    `id`         INT            NOT NULL AUTO_INCREMENT                COMMENT '設定ID',
    `key`        VARCHAR(100)   NOT NULL                               COMMENT '設定キー（例: target_cost_ratio）',
    `value`      VARCHAR(255)   NOT NULL                               COMMENT '設定値（文字列として格納）',
    `created_at` TIMESTAMP      NOT NULL DEFAULT CURRENT_TIMESTAMP     COMMENT '作成日時',
    `updated_at` TIMESTAMP      NOT NULL DEFAULT CURRENT_TIMESTAMP
                                         ON UPDATE CURRENT_TIMESTAMP   COMMENT '更新日時',
    PRIMARY KEY (`id`),
    UNIQUE KEY `uk_key` (`key`)
) ENGINE=InnoDB
  DEFAULT CHARSET=utf8mb4
  COLLATE=utf8mb4_unicode_ci
  COMMENT='システム設定マスタ';
```
